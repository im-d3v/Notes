Complexity Quick Reference

Time · Space · Gotchas — one table.

⏱️ Pattern → Time → Space → Notes

PatternTimeSpaceCritical Notes
Two PointersO(n)O(1)Requires sorted input. 3Sum = O(n²) due to outer loop.
Sliding WindowO(n)O(k)k = window size / charset. Each element enters & exits once.
Prefix SumO(n) build, O(1) queryO(n)HashMap variant for sum=k is O(n) time O(n) space.
Fast & SlowO(n)O(1)Cycle detection proof: Floyd's algorithm guarantees meeting.
Binary SearchO(log n)O(1)BS on answer space: O(n·log(maxVal)).
BFSO(V+E)O(V)Queue can hold all nodes at widest level.
DFSO(V+E)O(H)H = tree height. Worst case O(n) for skewed tree.
Merge IntervalsO(n log n)O(n)Sort dominates. Output list is the space cost.
BacktrackingO(2ⁿ) or O(n!)O(n)Pruning critical. Subsets=2ⁿ. Permutations=n!.
Top K ElementsO(n log k)O(k)Better than O(n log n) sort when k ≪ n.
Two HeapsO(n log n)O(n)Each insert/rebalance = O(log n).
Monotonic StackO(n)O(n)Each element pushed/popped at most once.
Topological SortO(V+E)O(V+E)Adjacency list + in-degree array. Cycle → no valid order.
Union FindO(α(n)) ≈ O(1)O(n)With path compression + union by rank. α = inverse Ackermann.
K-way MergeO(n log k)O(k)k lists in heap. n = total elements.
Cyclic SortO(n)O(1)In-place. Each element moved at most once.
Modified BSO(n log m)O(1)m = answer range. Feasibility check = O(n).
In-Place LL RevO(n)O(1)Three-pointer technique. No extra list needed.
DP 0/1 KnapsackO(n·W)O(W)Space optimized: 1D array, iterate W backwards.
DP Unbounded KSO(n·W)O(W)Same table but iterate W forwards (allow reuse).
DP FibonacciO(n)O(1)Two variables. No array needed.
DP LCSO(m·n)O(min(m,n))Space optimized to 1 row if reading row by row.
DP PalindromeO(n²)O(n²) or O(1)Expand-from-center: O(n²) time, O(1) space.
GreedyO(n log n)O(1)Sort dominates. Greedy step itself is O(n).
TrieO(L) per opO(N·L)L = word length. N = number of words.
Bitwise XORO(n)O(1)Constant extra space. XOR is associative + commutative.
Segment Tree / BITO(log n) update+queryO(n)BIT is simpler code; Seg Tree handles more query types.
DijkstraO((V+E) log V)O(V)Fails on negative weights. Use Bellman-Ford instead.

30 Common Traps — One Line Each

Quick-fire. Every single one has cost someone an offer.

💣 The 30 Traps — Memorize Before You Walk In

1
Two Pointers on unsorted input — sort first, otherwise answers are wrong.
2
Sliding Window with fixed K — don't shrink with while; just pop when size > k.
3
Prefix Sum off-by-one — pre[0]=0, query pre[r+1]-pre[l], not pre[r]-pre[l-1].
4
Binary Search — always check: is your invariant lo <= hi or lo < hi? Mix them → infinite loop.
5
BFS for shortest path — DFS gives A path, not THE shortest. Use BFS on unweighted graphs.
6
Forgetting to mark visited in BFS/DFS — graph problems loop forever without a visited set.
7
Backtracking — not removing the element from path before returning = incorrect result set.
8
DP 0/1 Knapsack — iterating capacity forwards (not backwards) causes item reuse = becomes Unbounded.
9
DP memoization — checking memo[i] == null fails when 0 is a valid answer; use memo[i] == -1 sentinel.
10
Greedy "feels right" but isn't — always ask: does a counterexample exist? DP is safer when unsure.
11
Topological Sort — if queue empties before all nodes are processed, there's a cycle in the graph.
12
Union Find without path compression — degrades to O(n) per operation. Always compress.
13
Fast & Slow — after finding cycle, reset one pointer to head to find entry point — don't forget this step.
14
Two Heaps rebalance — max-heap size must equal OR be one more than min-heap. Off-by-one corrupts median.
15
Merge Intervals — sorting by end instead of start causes missed merges.
16
Dijkstra on negative weights — wrong answers guaranteed. Use Bellman-Ford for negative edges.
17
Monotonic Stack — for "next greater element from the right", iterate right-to-left, not left-to-right.
18
Trie wildcard search — must recurse on all children when char is '.', not just one branch.
19
Cyclic Sort — values starting at 0 (not 1) → adjust index to a[i] = index directly, don't subtract 1.
20
LCS vs LIS — LCS compares two strings; LIS finds increasing subsequence in one array. Different DP.
21
Integer overflow in Binary Search — mid = lo + (hi - lo) / 2, never (lo + hi) / 2.
22
Sliding window char frequency — when count goes negative, don't stop; just let it go. Check formed count.
23
Graph DFS — forgetting to unmark visited during backtracking in permutation-style path problems.
24
K-way Merge — pushing null nodes to heap → NullPointerException. Always null-check before pushing.
25
DP palindrome table direction — must fill diagonally (by length), not row by row.
26
XOR to find two missing — x^y = xor_all; must use a set bit to partition into two groups. Don't skip step 2.
27
Backtracking subsets with duplicates — sort first AND skip nums[i] == nums[i-1] at same recursion depth.
28
BFS level tracking — must capture q.size() BEFORE the inner for-loop, not inside it.
29
In-place LL reversal — saving next before rewiring is mandatory; losing reference = infinite loop.
30
Modified Binary Search on answer — if feasible(mid) doesn't terminate correctly, BS loops. Verify boundary.

The Interview Ritual

10 questions. 10 seconds each. 90-second setup before you write a single line.

✅ The 90-Second Pre-Code Ritual

🎯
The Rule

Never write code before completing this checklist. Interviewers reward structured thinking over fast typing. Take the 90 seconds. It will save you 20 minutes of wrong-direction code.

1
What exactly is the input? What is the output? Repeat it back. Confirm types: int[]? String? List<List>? Graph? Tree?
2
What are the constraints? (n = ?) n ≤ 20 → brute/backtrack OK. n ≤ 10⁴ → O(n²) OK. n ≤ 10⁶ → must be O(n log n) or better.
3
Is the input sorted? Can I sort it? Sorting unlocks Two Pointers, Binary Search, Merge Intervals. Check if allowed.
4
Edge cases — what breaks naive solutions? Empty input, single element, all same, negative numbers, integer overflow, duplicates.
5
Which pattern does this look like? Run the flowchart mentally. Name the pattern out loud: "This looks like sliding window."
6
What is the brute-force solution first? Always state it. Even if O(n³). It shows you can solve it — then you optimize from there.
7
Where is the redundant work in brute force? What are you recomputing? That's exactly what the pattern eliminates.
8
What data structure do I need? HashMap for frequency/lookup. Heap for K-elements. Stack for matching/monotone. Queue for BFS.
9
What is the target time and space complexity? State it before coding: "I'm aiming for O(n log n) time and O(k) space."
10
Walk through a small example before coding. 2-3 elements. Trace manually. Catch logic errors BEFORE they become code bugs.

🏁 After Coding — 3 Final Checks

(A) Trace through your small example again with the actual code.   (B) Check all edge cases: empty, single, duplicates.   (C) Re-state the complexity — both time AND space.

Pattern Pairs — "These Look Similar But…"

The most common sources of wrong pattern selection in interviews.

🔀 The 5 Most Confused Pattern Pairs

PAIR 1A
👆 Two Pointers
  • Use when: Comparing elements from opposite ends
  • Input: Usually sorted (or must be sorted)
  • Window size: Not fixed — pointers move independently
  • Shrinks from: Both ends toward center
  • State: Two indices only, no frequency map needed
  • Signal words: "pair", "triplet", "sorted array", "palindrome check"
  • Example: LC#167 Two Sum II
PAIR 1B
🪟 Sliding Window
  • Use when: Processing a contiguous subarray/substring
  • Input: NOT required to be sorted
  • Window size: Fixed K or dynamic (shrink on violation)
  • Moves: Right pointer expands, left pointer shrinks
  • State: Frequency HashMap or running sum maintained
  • Signal words: "subarray", "substring", "contiguous", "at most K"
  • Example: LC#76 Min Window Substring
⚡ KEY DISTINCTION: Two Pointers = sorted + compare ends · Sliding Window = contiguous + maintain state
PAIR 2A
🌊 BFS
  • Use when: Finding shortest path or minimum steps
  • Structure: Queue (FIFO) — level by level
  • Explores: All neighbors at distance d before d+1
  • Guarantees: First path found = shortest path
  • Space: O(w) — w = max width of tree/graph
  • Signal words: "minimum steps", "shortest path", "level order"
  • Example: LC#127 Word Ladder
PAIR 2B
🌲 DFS
  • Use when: Exploring all paths or any valid path
  • Structure: Stack / recursion (LIFO) — depth first
  • Explores: One complete branch before backtracking
  • Guarantees: Finds A path — not necessarily shortest
  • Space: O(h) — h = max depth (recursion stack)
  • Signal words: "all paths", "connected", "cycle", "exists a path"
  • Example: LC#113 Path Sum II
⚡ KEY DISTINCTION: BFS = SHORTEST PATH · DFS = ALL PATHS / EXISTENCE
PAIR 3A
🔙 Backtracking
  • Use when: Generate ALL valid combinations/permutations
  • Pattern: Choose → recurse → unchoose (3-step ritual)
  • Pruning: Cut branches early if constraint already violated
  • State: Mutable path list that gets modified in-place
  • Output: A list of ALL valid results
  • Signal words: "all subsets", "all permutations", "generate all"
  • Example: LC#51 N-Queens
PAIR 3B
🌲 DFS (no backtrack)
  • Use when: Explore nodes, mark visited, check connectivity
  • Pattern: Visit → mark → recurse on unvisited neighbors
  • Pruning: Skip visited nodes (no "undo" of visited marking)
  • State: Boolean visited[] array, not modified on return
  • Output: A single answer (count, yes/no, distance)
  • Signal words: "connected components", "flood fill", "island count"
  • Example: LC#200 Number of Islands
⚡ KEY DISTINCTION: Backtracking UNDOES state on return · DFS keeps visited state permanent
PAIR 4A
🎒 Dynamic Programming
  • Use when: Overlapping subproblems + optimal substructure
  • Approach: Try ALL choices, memoize results
  • Guarantees: Global optimum always found
  • Cost: Higher time/space — fills full table
  • Proof: Not needed — enumerate and cache
  • Signal words: "max/min", "how many ways", "can you achieve"
  • Example: LC#72 Edit Distance
PAIR 4B
🥇 Greedy
  • Use when: Local optimum = global optimum (provable)
  • Approach: Make the BEST choice NOW, never revisit
  • Guarantees: Global optimum ONLY IF exchange argument holds
  • Cost: Lower — O(n log n) max, often O(n)
  • Proof: Must verify with exchange argument or counterexample
  • Signal words: "minimum cost", "maximum tasks", "intervals"
  • Example: LC#55 Jump Game
⚡ KEY DISTINCTION: DP = exhaustive + safe · Greedy = one-pass + needs proof · When unsure, use DP
PAIR 5A
🔗 Union Find
  • Use when: Dynamic connectivity, group membership queries
  • Input type: Undirected graph (edges can come in any order)
  • Cycle detect: Two nodes in same component = adding edge = cycle
  • Output: Is node A connected to node B? How many groups?
  • Direction: Does NOT care about direction of edges
  • Signal words: "connected", "same group", "merge", "cycle in undirected"
  • Example: LC#684 Redundant Connection
PAIR 5B
📋 Topological Sort
  • Use when: Ordering tasks with dependency constraints
  • Input type: Directed acyclic graph (DAG)
  • Cycle detect: If not all nodes processed = cycle exists
  • Output: A valid linear ordering of all nodes
  • Direction: REQUIRES directed edges — order matters
  • Signal words: "prerequisites", "order tasks", "build order", "DAG"
  • Example: LC#210 Course Schedule II
⚡ KEY DISTINCTION: Union Find = undirected connectivity · Topological Sort = directed ordering
🎓
The Meta-Rule That Wins Interviews

When two patterns feel equally valid: state both out loud. Say: "This could be DP or Greedy — let me check if greedy holds by seeing if a counterexample exists." That sentence alone signals senior-level thinking to your interviewer.

☕ Java Template Quick-Recall

Sliding Window (Variable)

JavaVariable Window
Map<Character, Integer> map = new HashMap<>();
int left = 0, res = 0;
for (int right = 0; right < s.length(); right++) {
    map.merge(s.charAt(right), 1, Integer::sum);
    while (invalid(map)) {
        map.merge(s.charAt(left++), -1, Integer::sum);
    }
    res = Math.max(res, right - left + 1);
}

Binary Search (Answer Space)

JavaBS on Answer
int lo = minAnswer(), hi = maxAnswer();
while (lo < hi) {
    int mid = lo + (hi - lo) / 2;
    if (feasible(mid)) {
        hi = mid;       // try smaller
    } else {
        lo = mid + 1;  // need bigger
    }
}
return lo;

DP 0/1 Knapsack (Space Optimized)

Java1D Knapsack
int[] dp = new int[W + 1];
for (int i = 0; i < n; i++) {
    // iterate BACKWARDS — prevents reuse
    for (int w = W; w >= wt[i]; w--) {
        dp[w] = Math.max(dp[w],
                    dp[w - wt[i]] + val[i]);
    }
}

Union Find (Path Compression)

JavaUnion Find
int[] parent = new int[n];
int[] rank   = new int[n];
// Arrays.fill(parent, 0..n-1)
int find(int x) {
    if (parent[x] != x)
        parent[x] = find(parent[x]);
    return parent[x];
}
void union(int a, int b) {
    int ra=find(a), rb=find(b);
    if(rank[ra]<rank[rb]) parent[ra]=rb;
    else if(rank[ra]>rank[rb]) parent[rb]=ra;
    else { parent[rb]=ra; rank[ra]++; }
}
