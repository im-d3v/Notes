The Selection Guide

Before writing a single line of code — pick the right algorithm.

🗺️ Graph Algorithm Selection Guide

💡
Mental Model — GPS Modes: BFS = walking on equal streets. Dijkstra = driving on roads with non-negative tolls. Bellman-Ford = GPS that handles negative toll rebates. Floyd-Warshall = pre-computing a full road atlas for every city pair. Kruskal/Prim = cheapest way to wire a city with power lines.
Problem Type Algorithm Why This One? Time When to Skip
Single source, unweighted BFS Each edge = cost 1. BFS explores level by level = distance by distance. O(V+E) Any edge has a distinct weight
Single source, weighted, no negatives Dijkstra Greedy relaxation is safe when all costs ≥ 0. Once popped, a node's distance is final. O((V+E) log V) Negative weights exist
Single source, weighted, negatives allowed Bellman-Ford Brute-force relaxes every edge N-1 times. Slower but correct for negatives. O(V × E) Graph is large (>10⁴ nodes)
All pairs shortest path Floyd-Warshall DP through every intermediate node. One pass builds the full distance matrix. O(V³) V > 500 (too slow)
Connect all nodes minimum cost Kruskal / Prim MST — minimum weight tree that spans all nodes. Not shortest path — connectivity. O(E log E) You need point-to-point shortest path
⚠️
The #1 interview mistake: Reaching for Dijkstra by reflex. Ask two questions before writing code: "Are there negative weights?" and "Do I need one source or all pairs?" Those two questions alone narrow it down completely.

Recognition Signals — What the problem says → What it means

Problem Says…Use
"minimum cost / time / delay to reach destination"Dijkstra
"at most K stops / hops / steps"Bellman-Ford (K+1 rounds) or modified Dijkstra with state
"city reachable within threshold distance"Floyd-Warshall (all pairs)
"connect all points / cities with minimum total cost"Kruskal or Prim (MST)
"currency arbitrage / negative cycle detection"Bellman-Ford Nth iteration check
"grid, minimum hops, no weights"BFS
02

Dijkstra's Algorithm

Greedy shortest path for non-negative weighted graphs.

⚡ Dijkstra's Algorithm — Deep Dive

The Mental Model

You're in a city. You start at node S. At every step, expand the cheapest unvisited city you know about. Use a min-heap so you always extract the minimum cost city first.

Once you pop a city from the heap, its distance is locked forever. No future path through positive edges can beat it. This is the greedy invariant that makes Dijkstra correct and fast.

🔑

Core Invariant: When a node is popped from the min-heap, its distance is final. All positive-weight future paths can only be longer — so we never revisit a popped node.

Java Template — Dijkstra with Min-Heap

Java Dijkstra — Standard Priority Queue Template 
import java.util.*;

class Dijkstra {

    // graph.get(u) = list of int[]{neighbor, weight}
    public static int[] shortestPaths(
            List<List<int[]>> graph, int src, int n) {

        int[] dist = new int[n];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        // Min-heap ordered by distance: {distance, node}
        PriorityQueue<int[]> pq = new PriorityQueue<>(
                (a, b) -> a[0] - b[0]);
        pq.offer(new int[]{0, src});

        while (!pq.isEmpty()) {
            int[] curr = pq.poll();
            int d = curr[0], u = curr[1];

            // ✅ Stale entry check — critical, do not skip
            if (d > dist[u]) continue;

            for (int[] edge : graph.get(u)) {
                int v = edge[0], w = edge[1];
                if (dist[u] + w < dist[v]) {      // relaxation
                    dist[v] = dist[u] + w;
                    pq.offer(new int[]{dist[v], v});
                }
            }
        }
        return dist;   // dist[i] == Integer.MAX_VALUE → unreachable
    }
}
🚫
Why Dijkstra FAILS with Negative Edges

Suppose A→B = 4, but A→C→B = 3 + (−5) = −2. Dijkstra pops B at cost 4 and locks it. It never reconsiders. The true shortest path −2 is silently ignored. Negative edges break the invariant that "popped = optimal."

Complexity at a Glance

ImplementationTimeSpaceInterview Use
Binary min-heap (PriorityQueue) O((V+E) log V) O(V+E) Default choice
Linear array scan O(V²) O(V) Dense graph (E ≈ V²)
Fibonacci heap O(E + V log V) O(V) Never in interviews

Acceptable in practice: up to ~10⁵ nodes and ~10⁶ edges with the min-heap version.

🎯
Decision Moment — Dijkstra vs BFS: If all edges have the same weight (or weight = 1), use BFS — it's O(V+E) vs O((V+E) log V). Only upgrade to Dijkstra when edge weights genuinely differ.

03

Bellman-Ford Algorithm

Slower but universal — handles negatives and detects negative cycles.

🌀 Bellman-Ford — Relax ALL Edges, N-1 Times

The Mental Model

Don't trust greedy. Instead, brute-force it: "The shortest path with at most 1 edge? Relax once. With at most 2 edges? Relax twice..."

A simple path in a graph with V nodes uses at most V-1 edges. So relax all edges exactly N-1 times. After that, any further improvement signals a negative cycle — a loop you can keep traversing to decrease cost infinitely.

🔑

Why N-1 iterations? After i rounds, dist[v] holds the shortest path using at most i edges. A graph with V nodes has paths of at most V-1 edges. So N-1 rounds = complete.

Java Bellman-Ford — Standard Template + Negative Cycle Detection 
class BellmanFord {

    // edges[i] = {from, to, weight}
    public static int[] shortestPaths(
            int[][] edges, int n, int src) {

        int[] dist = new int[n];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        // N-1 relaxation rounds
        for (int i = 0; i < n - 1; i++) {
            boolean updated = false;
            for (int[] e : edges) {
                int u = e[0], v = e[1], w = e[2];
                if (dist[u] != Integer.MAX_VALUE
                        && dist[u] + w < dist[v]) {
                    dist[v] = dist[u] + w;
                    updated = true;
                }
            }
            if (!updated) break; // early exit — already converged
        }

        // Nth round: if anything still relaxes → negative cycle
        for (int[] e : edges) {
            int u = e[0], v = e[1], w = e[2];
            if (dist[u] != Integer.MAX_VALUE
                    && dist[u] + w < dist[v]) {
                throw new IllegalStateException("Negative cycle!");
            }
        }

        return dist;
    }
}

LC #787 — The K-Stops Variant (Classic Bellman-Ford Trap)

Run exactly K+1 relaxation rounds (K stops = K+1 edges). The critical trap is using a snapshot of the distance array at the start of each round.

🚫
Without the snapshot: Within a single round you might "chain" relaxations — using an edge relaxed earlier in the same round. This lets a single iteration take a path of 3+ edges, breaking the "at most K edges" constraint silently.
Java LC #787 — Cheapest Flights Within K Stops 
public int findCheapestPrice(
        int n, int[][] flights, int src, int dst, int k) {

    int[] dist = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[src] = 0;

    for (int i = 0; i <= k; i++) {               // K+1 rounds
        int[] snap = Arrays.copyOf(dist, n);         // ✅ SNAPSHOT
        for (int[] f : flights) {
            int u = f[0], v = f[1], w = f[2];
            if (dist[u] != Integer.MAX_VALUE
                    && dist[u] + w < snap[v]) {
                snap[v] = dist[u] + w;
            }
        }
        dist = snap;
    }

    return dist[dst] == Integer.MAX_VALUE ? -1 : dist[dst];
}

Bellman-Ford vs Dijkstra — Decision Table

ConditionWinnerReason
All weights ≥ 0, no hop limitDijkstraFaster: O((V+E) log V)
Any negative weight possibleBellman-FordGreedy invariant breaks with negatives
"At most K hops" constraintBellman-FordRun K+1 rounds with snapshot
Need to detect negative cycleBellman-FordRun Nth round, check for any update
Graph > 10⁴ nodesDijkstraBellman-Ford too slow at scale

Floyd-Warshall Algorithm

All-pairs shortest paths — when you need every-to-every distance at once.

🔷 Floyd-Warshall — Try Every Node as an Intermediate

The Mental Model

Ask this for every pair (i, j): "Is it shorter to go i→j directly, or i→k→j through some intermediate node k?"

Iterate over every possible intermediate node k from 0 to N-1. After all N iterations, dist[i][j] holds the true shortest path. It's DP where you gradually allow more intermediates.

🚫
Loop Order is Sacred — the most common Floyd-Warshall bug:

The outer loop MUST be k (the intermediate node). If you put k in the inner loop, you'll compute dist[i][j] through k before dist[i][k] is fully optimized, giving wrong answers. Always: for k → for i → for j.

Java Floyd-Warshall — All Pairs Shortest Path 
class FloydWarshall {

    public static int[][] allPairs(int n, int[][] edges) {
        final int INF = (int) 1e9;
        int[][] dist = new int[n][n];

        // 1. Init — INF everywhere, 0 on diagonal
        for (int[] row : dist) Arrays.fill(row, INF);
        for (int i = 0; i < n; i++) dist[i][i] = 0;

        // 2. Load direct edges
        for (int[] e : edges) {
            dist[e[0]][e[1]] = e[2];
            dist[e[1]][e[0]] = e[2]; // remove for directed graph
        }

        // 3. OUTER = k (intermediate), then i, then j — order is non-negotiable
        for (int k = 0; k < n; k++) {
            for (int i = 0; i < n; i++) {
                for (int j = 0; j < n; j++) {
                    if (dist[i][k] < INF && dist[k][j] < INF) {
                        dist[i][j] = Math.min(
                            dist[i][j],
                            dist[i][k] + dist[k][j]
                        );
                    }
                }
            }
        }

        // Negative cycle: if dist[i][i] < 0 for any i → cycle exists
        return dist;
    }
}

When to Use Floyd-Warshall — Checklist

  • ✅ You need distances between every pair of nodes
  • ✅ Graph is small: V ≤ 200 (safe), V ≤ 400 (borderline), V > 500 (TLE)
  • ✅ Query asks "how many nodes are reachable from X within threshold?" — needs full matrix
  • ❌ You only need shortest path from one source → use Dijkstra
  • ❌ Graph is large → run Dijkstra from every node instead (V × Dijkstra)

05

Minimum Spanning Tree

Connect all nodes — minimum total weight, zero cycles.

🌲 Kruskal's & Prim's Algorithms

What Is a Spanning Tree?

A spanning tree of V nodes: connects all V nodes, has exactly V-1 edges, no cycles. A Minimum Spanning Tree (MST) minimizes the total edge weight sum.

⚠️
MST ≠ Shortest Path Tree. The MST gives minimum cost to connect every node. It does NOT give shortest paths between individual pairs. These are fundamentally different goals — don't confuse them.

🔵 Kruskal's

Strategy: Sort all edges by weight. Add cheapest edge that doesn't create a cycle. Use Union-Find to detect cycles in near-O(1).

Best when: Sparse graph. Edge list is the natural input format.

1
Sort all edges by weight (ascending)
2
For each edge: union(u,v) — if different components, add it
3
Stop when V-1 edges added

🟢 Prim's

Strategy: Start from any node. Greedily expand the tree by adding the cheapest edge crossing the cut (tree ↔ non-tree boundary).

Best when: Dense graph. Adjacency list / matrix is the natural input.

1
Add node 0 to MST, push its edges to min-heap
2
Pop cheapest edge to unvisited node → add to tree
3
Push new node's edges. Repeat until all V nodes in MST.

Java Template — Kruskal's with Union-Find (DSU)

Java Kruskal's MST — DSU with Path Compression + Union by Rank 
class Kruskal {

    int[] parent, rank;

    int find(int x) {
        if (parent[x] != x)
            parent[x] = find(parent[x]);  // path compression
        return parent[x];
    }

    boolean union(int x, int y) {
        int px = find(x), py = find(y);
        if (px == py) return false;           // same component → cycle!
        if (rank[px] < rank[py]) { int t = px; px = py; py = t; }
        parent[py] = px;
        if (rank[px] == rank[py]) rank[px]++;
        return true;
    }

    public int mstCost(int n, int[][] edges) {
        parent = new int[n];
        rank   = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;

        Arrays.sort(edges, (a, b) -> a[2] - b[2]);  // sort by weight

        int totalCost = 0, used = 0;
        for (int[] e : edges) {
            if (union(e[0], e[1])) {
                totalCost += e[2];
                if (++used == n - 1) break;   // MST complete
            }
        }
        return used == n - 1 ? totalCost : -1; // -1 → disconnected
    }
}

Java Template — Prim's with Min-Heap

Java Prim's MST — Priority Queue Template 
public int primMST(int n, List<List<int[]>> graph) {
    boolean[] inMST = new boolean[n];
    // min-heap: {edge_weight, destination_node}
    PriorityQueue<int[]> pq = new PriorityQueue<>(
            (a, b) -> a[0] - b[0]);
    pq.offer(new int[]{0, 0}); // start from node 0, cost 0

    int totalCost = 0, added = 0;

    while (!pq.isEmpty() && added < n) {
        int[] curr = pq.poll();
        int cost = curr[0], node = curr[1];

        if (inMST[node]) continue;     // already in tree
        inMST[node] = true;
        totalCost += cost;
        added++;

        for (int[] nb : graph.get(node)) {
            if (!inMST[nb[0]])
                pq.offer(new int[]{nb[1], nb[0]});
        }
    }
    return added == n ? totalCost : -1;
}

Kruskal vs Prim — When to Pick Which

ConditionWinnerReason
Sparse graph (E ≈ V)KruskalSorting E edges is cheap; Union-Find nearly O(1)
Dense graph (E ≈ V²)PrimNo need to sort all E² edges; heap approach is faster
Input is an edge listKruskalNaturally operates on edge list
Input is adjacency list/matrixPrimNaturally expands from adjacency

Common Interview Traps

The mistakes that separate "solved it" from "thought you solved it."

🕳️ Graph Algorithm — The 5 Traps That Kill Interviews

TRAP 1 — DIRECTED VS UNDIRECTED CYCLE DETECTION

In an undirected graph, the edge A-B means you can traverse both A→B and B→A. DFS will see the parent as a "visited" neighbor — that is NOT a cycle. You must track and skip the parent node.

In a directed graph, you need a separate in-stack boolean array — only a back-edge to a node currently on the DFS call stack is a true cycle. "Visited" alone is insufficient.

JavaUndirected vs Directed Cycle Detection
// Undirected: skip the node we came from (parent), not all visited
boolean hasCycleUndirected(int node, int parent, boolean[] vis) {
    vis[node] = true;
    for (int nei : graph[node]) {
        if (!vis[nei]) {
            if (hasCycleUndirected(nei, node, vis)) return true;
        } else if (nei != parent) return true; // back edge = cycle
    }
    return false;
}

// Directed: track which nodes are on the CURRENT dfs path
boolean hasCycleDirected(int node, boolean[] vis, boolean[] inStack) {
    vis[node] = inStack[node] = true;
    for (int nei : graph[node]) {
        if (!vis[nei] && hasCycleDirected(nei, vis, inStack)) return true;
        if (inStack[nei]) return true; // back edge to current path
    }
    inStack[node] = false; // remove from path on way back
    return false;
}
TRAP 2 — DISCONNECTED GRAPH HANDLING

Dijkstra/BFS from a single source will NOT visit disconnected components. If the problem asks "can every node be reached?" or "return the max distance," you need to check for unreachable nodes after the algorithm.

🚫
Never return Integer.MAX_VALUE as a distance. Always convert: if (dist[target] == Integer.MAX_VALUE) return -1;

For undirected graphs, if you need to visit all components, loop through all unvisited nodes and start a new BFS/DFS from each one.

TRAP 3 — OVERFLOW WHEN CHECKING DIST[U] + W

If dist[u] = Integer.MAX_VALUE (unreachable node), then dist[u] + w overflows to a large negative number, causing spurious relaxations that corrupt the entire distance array.

Fix: Always guard: if (dist[u] != Integer.MAX_VALUE && dist[u] + w < dist[v])

TRAP 5 — MST vs SHORTEST PATH CONFUSION

Students confuse LC #1584 (MST — connect all points cheaply) with LC #743 (Dijkstra — shortest delay to all nodes). The MST minimizes total cable cost across all connections. Dijkstra minimizes path length from a specific source. They solve completely different problems and produce different trees.

✅ MST

  • No specific source node
  • Connects ALL nodes with minimum total weight
  • Result: a tree with V-1 edges
  • Example: laying fiber optic cable

📍 Shortest Path

  • Has a specific source node
  • Minimum distance from source to each node
  • Result: shortest path tree (may use more edges)
  • Example: GPS navigation from your location
