Foundations

What sliding window is, why it works, and the critical Fixed vs Variable distinction.

🪟 What Is the Sliding Window Pattern?

Imagine a physical frame placed over an array. The frame has a left edge and a right edge. Everything inside the frame is your current window. Instead of recomputing the window from scratch each time you move it, you add the new element entering on the right and remove the old element leaving on the left. One slide forward costs O(1) — not O(k).

This converts the classic nested-loop O(n × k) approach into a single O(n) scan.

The key insight

new_sum = old_sum − arr[left−1] + arr[right] — we never loop through the window again. Each element is touched at most twice: once when the right pointer adds it, once when the left pointer removes it. Total work = O(2n) = O(n).

⚡ Fixed Window vs Variable Window — The #1 Confusion

⚠️
This is the most common source of bugs in sliding window solutions.

Choosing the wrong variant leads to either missing valid windows or checking too many. Read the decision rule carefully before writing a single line of code.

✅ Fixed Window

The window size never changes. Left and right pointers always move together — right expands by 1, left shrinks by 1, maintaining constant gap.


Signature: problem gives you a number k or size.


Examples:

  • Max sum of k consecutive elements
  • First negative in every window of k
  • Find all anagrams of length k

🔄 Variable Window

The window grows and shrinks based on a condition. Right always moves forward; left moves only when the window becomes invalid.


Signature: problem asks for longest/shortest/minimum/maximum window satisfying a condition.


Examples:

  • Longest substring without repeating characters
  • Minimum window containing all chars of T
  • Longest subarray with sum ≤ target

🎯 The Decision Rule

❓ Is the window size fixed by the problem?
YES → Fixed Window

Problem says "of size k", "consecutive k elements", "every window of length k".

Use the fixed template — slide left+right together.

NO → Variable Window

Problem says "longest", "shortest", "minimum", "maximum", "at most k", "exactly k distinct".

Use the variable template — right always moves, left conditionally shrinks.

🎙️
Interview tip — say this out loud

"The problem asks for the longest window satisfying X, so this is a variable sliding window. I'll expand right freely and shrink from left whenever the window becomes invalid." — Saying this before coding signals pattern mastery.

02

Core Templates

Production-ready templates — memorise these, adapt to any problem.

📏 Fixed Window Template

Use this whenever the problem gives you an explicit window size k.

Java Fixed Window — Max Sum of K Elements 
public int maxSumFixedWindow(int[] arr, int k) {
    int n = arr.length;

    // ── GUARD: not enough elements ────────────────────────
    if (n < k) return -1;

    // ── PHASE 1: build the initial window of size k ────────
    int windowSum = 0;
    for (int i = 0; i < k; i++) {
        windowSum += arr[i];          // load first k elements
    }
    int maxSum = windowSum;

    // ── PHASE 2: slide the window across the rest ──────────
    for (int right = k; right < n; right++) {
        windowSum += arr[right];      // add new element entering on RIGHT
        windowSum -= arr[right - k];  // remove old element leaving on LEFT
        // left pointer is implicitly (right - k + 1)
        maxSum = Math.max(maxSum, windowSum);
    }

    return maxSum;
}

Fixed Window Mental Model

Think of it as a conveyor belt of width k. The belt moves one slot right each iteration: one item drops off the back, one item climbs on the front. You never rebuild the belt from scratch.

🔄 Variable Window Template — Fully Annotated

ℹ️
This is the heart of the pattern.

Every variable sliding window problem is a specialisation of this template. The only things that change are: (1) the validity condition, (2) what state you track, (3) whether you want max or min window.

Java Variable Window — Universal Template (annotated) 
public int variableWindow(int[] arr) {
    int left = 0;        // ① LEFT pointer — marks window start, moves inward to shrink
    int best = 0;        // ② ANSWER accumulator (could be 0, INT_MAX, etc.)
    int state = 0;      // ③ WINDOW STATE — tracks sum / count / freq map, etc.

    for (int right = 0; right < arr.length; right++) {

        // ④ EXPAND — right pointer moves forward every iteration (always)
        state = update(state, arr[right]);

        // ⑤ SHRINK LOOP — contract from left while window is INVALID
        //    "invalid" = violates the problem's condition
        //    The while loop (not if!) handles cases where multiple
        //    left advances are needed to restore validity.
        while (isInvalid(state)) {       // e.g. sum > target, distinct > k
            state = remove(state, arr[left]);
            left++;                        // shrink from LEFT
        }

        // ⑥ RECORD — window [left..right] is now valid; check if it's the best
        //    For LONGEST: best = max(best, right - left + 1)
        //    For SHORTEST: if valid, best = min(best, right - left + 1)
        best = Math.max(best, right - left + 1);
    }

    return best;
}

right = 4 (arr[4] = 1)
Expand: state = 4 + 1 = 5. Is 5 > 5? No → skip shrink.
Window [2..4] = {3,1,1}, sum=5, best=3 ← answer

🤔 When to Shrink the Window — The Intuition

🛋️ The Studio Apartment Analogy

Imagine you're packing furniture into a studio apartment (the window). You keep bringing in new furniture from the front door (right pointer expands). The apartment has a rule: you can only have items worth < $500 total. The moment a new item pushes you over $500, you start carrying older items out the back door (left pointer shrinks) — one by one — until you're back under $500. You never stop bringing in new furniture from the front. You only evict from the back when forced to. The largest arrangement you had that fit is your answer.

The while vs if question

Use while (isInvalid) — not if — because removing one element might not be enough to restore validity. For example, if your window has sum 50 and target is 10, you might need to evict many elements from the left before the sum drops below 10.

The while loop automatically handles this: it keeps evicting until the window is valid again, then stops immediately.

Shrink condition for LONGEST

Shrink when window becomes invalid. Record window size after shrinking (when valid again).

Java
while (invalid) {
    remove(arr[left]);
    left++;
}
best = max(best, right-left+1);

Shrink condition for SHORTEST

Shrink as long as window stays valid. Record window size before shrinking breaks validity.

Java
while (valid) {
    best = min(best, right-left+1);
    remove(arr[left]);
    left++;
}

Advanced Patterns

HashMap frequency tracking and the Sliding Window vs Two Pointers clarification.

🗺️ HashMap + Sliding Window — Character Frequency Tracking

40% of sliding window problems involve tracking character/element frequencies. The pattern: maintain a HashMap<Character, Integer> representing the current window's frequency table. Expand right by incrementing the count, shrink left by decrementing (and removing the key when count hits 0).

Java Longest Substring Without Repeating Characters — LC #3 
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> freq = new HashMap<>();
    int left = 0, best = 0;

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);

        // ① ADD incoming character to frequency map
        freq.put(c, freq.getOrDefault(c, 0) + 1);

        // ② SHRINK while a duplicate exists in the window
        while (freq.get(c) > 1) {   // only c can cause a duplicate
            char leftChar = s.charAt(left);
            freq.put(leftChar, freq.get(leftChar) - 1);
            if (freq.get(leftChar) == 0) freq.remove(leftChar); // cleanup
            left++;
        }

        // ③ RECORD window size — no duplicate in [left..right]
        best = Math.max(best, right - left + 1);
    }
    return best;
}

Optimisation tip

For ASCII strings, replace HashMap with int[128] (or int[256]). Array access is O(1) with zero hashing overhead. For Unicode use the map.

🆚 Sliding Window vs Two Pointers — The Clear Distinction

These two patterns share the same two-pointer mechanics but serve completely different purposes. Confusing them leads to applying the wrong mental model.

Dimension 🪟 Sliding Window 👉 Two Pointers
Core idea Maintain a contiguous subarray/substring window; everything between the pointers matters Navigate from two sides of an array; the space between pointers is usually discarded
Pointer movement Right always advances; left only advances to shrink Both pointers converge (or one chases the other) based on comparison logic
What you process The interior of the window — the contiguous block The pair pointed to by left and right
Data structure Often needs auxiliary state (HashMap, running sum, counter) Usually just the two index values — no auxiliary needed
Works on Unsorted or sorted arrays/strings Usually requires sorted input
Classic problems Longest substring without repeat, min window substring, max sum subarray Two Sum (sorted), Container With Most Water, 3Sum, Palindrome check
Keyword in problem "Subarray", "substring", "contiguous", "window" "Pair", "complement", "sorted array", "palindrome", "remove duplicates"

🪟 Sliding Window thinking

"I care about the entire contents of my window."

  • Tracking a sum of all elements inside
  • Checking if window contains all chars of T
  • Finding the longest valid stretch

👉 Two Pointers thinking

"I only care about the values at the two pointers."

  • Does arr[L] + arr[R] == target?
  • Is arr[L] == arr[R] a palindrome match?
  • Move L right or R left based on comparison
🧠

One-line test: Do you need to know what's inside the window? → Sliding Window. Do you only care about what's at the two endpoints? → Two Pointers.

Deep Walkthrough

LC #76 — the hardest commonly asked sliding window problem. Master this, master the pattern.

🔬 LC #76 — Minimum Window Substring

🔥
This is the boss-level sliding window problem.

If you can solve this confidently in an interview, you have genuinely mastered variable sliding windows. It combines HashMap frequency tracking, a validity counter trick, and shortest-window logic all at once.

1
Understand the Problem

Given strings s and t, return the minimum window substring of s such that every character in t (including duplicates) is included. Return "" if no such window exists.

Example: s = "ADOBECODEBANC", t = "ABC" → answer = "BANC"

Key constraint: duplicates in t must all be satisfied. If t = "AAB", the window must contain at least 2 A's and 1 B.

2
The Validity Counter Trick

Naively checking if all chars of T are in the window requires iterating the entire freq map every step — that's O(∣Σ∣) per step, not O(1).

Instead: maintain an integer have (characters fully satisfied) and need (total distinct characters required). A character is "satisfied" when its window count reaches its required count in T.

Window is valid when have == need. Checking validity is now O(1).

3
Complete Annotated Solution
Java LC #76 — Minimum Window Substring 
public String minWindow(String s, String t) {

    // ① Build the requirement map from T
    Map<Character, Integer> need = new HashMap<>();
    for (char c : t.toCharArray())
        need.put(c, need.getOrDefault(c, 0) + 1);

    Map<Character, Integer> window = new HashMap<>();
    int have = 0;                    // chars FULLY satisfied in current window
    int required = need.size();       // distinct chars we need to satisfy

    int left = 0;
    int[] best = {-1, 0, 0};          // [window length, left, right]

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);

        // ② EXPAND — add right char to window
        window.put(c, window.getOrDefault(c, 0) + 1);

        // ③ Check if this char's requirement is NOW satisfied
        if (need.containsKey(c) &&
            window.get(c).intValue() == need.get(c).intValue()) {
            have++;                    // one more char type fully covered
        }

        // ④ SHRINK LOOP — only when window is valid (have == required)
        while (have == required) {

            // ⑤ RECORD minimum window found so far
            int len = right - left + 1;
            if (best[0] == -1 || len < best[0]) {
                best[0] = len;
                best[1] = left;
                best[2] = right;
            }

            // ⑥ SHRINK from left — remove leftmost char
            char leftChar = s.charAt(left);
            window.put(leftChar, window.get(leftChar) - 1);

            // Did shrinking break a satisfied requirement?
            if (need.containsKey(leftChar) &&
                window.get(leftChar) < need.get(leftChar)) {
                have--;                // one char type no longer fully covered
            }
            left++;
        }
    }

    return best[0] == -1 ? ""
         : s.substring(best[1], best[2] + 1);
}

Separated by window type. Solve them in order — difficulty increases within each group.

📏 Fixed Window Problems

📌
How to approach fixed window problems

The window size k is always given. Build the initial window in Phase 1, then slide in Phase 2. These are typically easier and good for building the muscle memory.

  • #643 Maximum Average Subarray I
    Classic fixed window — find subarray of length k with max average     Easy
  • #1343 Sub-arrays of Size K ≥ Threshold
    Count windows with average ≥ threshold — good for counting pattern     Medium
  • #567 Permutation in String
    Fixed window = length of p2. Check if window is anagram of p1 (freq map comparison)     Medium
  • #438 Find All Anagrams in a String
    Extension of #567 — collect all valid starting indices     Medium
  • #1456 Maximum Vowels in Substring of Length K
    Fixed window with simple element type check     Medium

🔄 Variable Window Problems

⚠️
Variable windows require more thought

Before coding, always identify: (1) What is the validity condition? (2) What state do I need to track? (3) Am I looking for max or min window? Writing these 3 answers on paper before typing saves 80% of debugging time.

  • #121 Best Time to Buy and Sell Stock
    Warm-up — track running minimum (degenerate window)     Easy
  • #3 Longest Substring Without Repeating Characters
    The canonical variable window + HashMap problem. Must know this cold.     Medium
  • #209 Minimum Size Subarray Sum
    Shortest window with sum ≥ target — shrink when valid (inverse pattern)     Medium
  • #1004 Max Consecutive Ones III
    Longest subarray with at most k zeros. Shrink when zeros > k.     Medium
  • #424 Longest Repeating Character Replacement
    Track max frequency in window. Shrink when (window size - maxFreq) > k     Medium
  • #1838 Frequency of Most Frequent Element
    Sort + variable window. Challenging validity condition involving total ops.     Medium
  • #76 Minimum Window Substring
    The boss — full walktrough above. Solve this last.     Hard
  • #30 Substring with Concatenation of All Words
    Sliding window where each unit is a word (not a char). Multiple starting offsets.     Hard
