The Framework — How to Think

Stop jumping to code. Start designing. This part teaches you the systematic thinking process that separates senior engineers from juniors in LLD rounds.

🎯 What Interviewers Actually Evaluate

🚩
The #1 Red Flag

Opening an IDE and starting to type a class within the first 2 minutes. This tells the interviewer you don't know how to design — you only know how to code.

✅
The Green Flag Answer

Talking through the design for 5–10 minutes before writing a single line. Walking the interviewer through your thinking is the entire point of the exercise.

The 7 Evaluation Axes

Every LLD interviewer at Google/Meta/Amazon scores you on these dimensions — whether or not they tell you that explicitly:

1
Clarify Requirements (2–3 minutes)

Ask functional questions first, then constraint questions. Write down the use cases as a numbered list. You'll trace every design decision back to them.

Functional Questions

  • What are the core use cases?
  • Who are the actors? (User, Admin, System)
  • What is explicitly OUT of scope?
  • What does success look like for the user?

Constraint Questions

  • Scale? (usually ignored in LLD — confirm)
  • Any specific patterns required?
  • Language preference?
  • Do we need persistence / DB?
2
Identify Entities (3–4 minutes)

Read the problem statement. Underline every NOUN → potential class. Underline every VERB → potential method.

Example — Parking Lot Problem

Problem: "A parking lot has multiple floors. Vehicles enter through an entrance gate, get a ticket, park in an available spot. On exit they pay a fee and the spot is freed." NOUNS (entities): ParkingLot · ParkingFloor · ParkingSpot Vehicle · Ticket · Payment · Gate · Fee VERBS (methods): park() · unpark() · calculateFee() issueTicket() · processPayment() · checkAvailability() ELIMINATE: "available" → attribute of Spot, not a class "fee" → could just be a double field on Ticket
⚠️
Over-modeling trap

You should have 5–10 core entities. If you have 15+, you're modeling attributes as classes. Ask yourself: "Does this noun have behavior? Or is it just data?"

3
Define Relationships (3–4 minutes)
RelationshipMeaningExampleWhen to Use
IS-A (Inheritance) Child is a specialized type of parent Car IS-A Vehicle Use Sparingly — most things are NOT is-a
HAS-A (Composition) Child cannot exist without parent Spot HAS-A Floor (dies with it) Prefer This — most relationships
HAS-A (Aggregation) Child exists independently of parent Vehicle exists without ParkingLot Looser coupling
USES-A (Dependency) Method-level, temporary use Gate USES-A TicketService Short-lived interactions
💡
Composition vs Aggregation — the death test

Ask: "If the parent is destroyed, does the child also cease to exist?" If yes → Composition. If no → Aggregation. A ParkingSpot without a Floor makes no sense → Composition. A Vehicle without a ParkingLot is still a vehicle → Aggregation.

4
Identify Abstractions (2–3 minutes)

For every entity, ask: "Will there be multiple concrete types of this?" If yes → extract an interface or abstract class.

Use Interface when…

  • No shared implementation
  • "Can do" relationship
  • Multiple inheritance needed
  • Examples: Payable, Observable, FeeCalculationStrategy

Use Abstract Class when…

  • Shared code exists across types
  • "Is a" relationship + common state
  • You want template method pattern
  • Examples: Vehicle, Piece (Chess)
5
Define the API / Key Methods (2 minutes)

Before writing any class body, define what each main class exposes. This forces interface-first thinking and gives you a map before coding.

TextAPI-first design — Parking Lot example
ParkingLot:
  + parkVehicle(vehicle: Vehicle): Ticket
  + unparkVehicle(ticket: Ticket): Payment
  + getAvailableSpots(type: VehicleType): List<ParkingSpot>

ParkingSpot:
  + assignVehicle(vehicle: Vehicle): void
  + removeVehicle(): void
  + isAvailable(): boolean
  + getSpotType(): SpotType

Ticket:
  + getIssuedTime(): LocalDateTime
  + getVehicle(): Vehicle
  + getSpot(): ParkingSpot
6
Code — Always in This Order
🚫
Do NOT start with the Manager/Service class or the main() method

This is the single most common LLD interview mistake. You end up with a God class that has no foundation. Always build bottom-up.

  1. Enums first — VehicleType, SpotType, PaymentStatus
  2. Core model classes — plain data classes (Vehicle, Ticket)
  3. Interfaces and Abstract classes — the contracts
  4. Concrete implementations — Car, HourlyFeeStrategy
  5. Manager/Service classes — the orchestrators (ParkingLotManager)
  6. Entry point/demo — main method or JUnit test

🔧 Design Patterns — LLD Quick Reference

💡
Key Insight

Don't memorize all 23 GoF patterns. Know these 8 deeply — they cover 95% of LLD interview problems. For each: understand the PROBLEM it solves, not just the structure.

Singleton

Problem it solves: Some things should only ever have one instance — a second instance would be a bug. Think database connection pools, configuration managers, logging systems.

LLD Uses

  • ParkingLotManager
  • ElevatorController
  • Logger
  • ConfigManager

⚠️ Common Trap

Don't Singleton everything. Justify it: "There should logically be only one ParkingLotManager — having two would mean split inventories."

JavaThread-safe Singleton — Double Checked Locking
public class ParkingLotManager {
    private static volatile ParkingLotManager instance;

    private ParkingLotManager() {} // prevent external instantiation

    public static ParkingLotManager getInstance() {
        if (instance == null) {                        // first check (no lock)
            synchronized (ParkingLotManager.class) {
                if (instance == null) {                  // second check (with lock)
                    instance = new ParkingLotManager();
                }
            }
        }
        return instance;
    }
}

Strategy

Problem it solves: You have one action (e.g., calculate fee) but multiple algorithms for it. You need to swap algorithms at runtime without changing the caller.

JavaStrategy Pattern — Fee Calculation
// The contract — define algorithm interface
public interface FeeCalculationStrategy {
    double calculateFee(Ticket ticket);
}

// Concrete strategy 1 — charges per hour
public class HourlyFeeStrategy implements FeeCalculationStrategy {
    private final double ratePerHour;

    public HourlyFeeStrategy(double ratePerHour) {
        this.ratePerHour = ratePerHour;
    }

    public double calculateFee(Ticket ticket) {
        long hours = ChronoUnit.HOURS.between(ticket.getIssuedTime(), LocalDateTime.now());
        return Math.max(1, hours) * ratePerHour; // minimum 1 hour charge
    }
}

// Concrete strategy 2 — flat daily pass
public class DayPassFeeStrategy implements FeeCalculationStrategy {
    private final double dayPassRate;

    public double calculateFee(Ticket ticket) {
        return dayPassRate; // flat fee regardless of duration
    }
}

// Context — accepts any strategy, doesn't care which
public class ExitGate {
    private FeeCalculationStrategy feeStrategy; // ← interface, NOT concrete class

    public ExitGate(FeeCalculationStrategy strategy) {
        this.feeStrategy = strategy;
    }

    public Payment processExit(Ticket ticket) {
        double fee = feeStrategy.calculateFee(ticket); // delegates to strategy
        return new Payment(fee, PaymentMode.CASH);
    }
}

Observer

Problem it solves: When one object changes state, multiple other objects need to react — but you don't want the subject to know who they are (loose coupling).

JavaObserver Pattern — Parking Spot Availability Display
// Subject interface — what objects can be observed
public interface ParkingObservable {
    void addObserver(ParkingObserver observer);
    void removeObserver(ParkingObserver observer);
    void notifyObservers();
}

// Observer interface — what all listeners must implement
public interface ParkingObserver {
    void update(ParkingFloor floor, int availableSpots);
}

// Subject implementation — floor notifies boards when spots change
public class ParkingFloor implements ParkingObservable {
    private List<ParkingObserver> observers = new ArrayList<>();
    private List<ParkingSpot> spots;

    public void addObserver(ParkingObserver obs) { observers.add(obs); }
    public void removeObserver(ParkingObserver obs) { observers.remove(obs); }

    public void notifyObservers() {
        long available = spots.stream().filter(ParkingSpot::isAvailable).count();
        observers.forEach(obs -> obs.update(this, (int) available));
    }
}

// Concrete observer — display board reacts to changes
public class DisplayBoard implements ParkingObserver {
    public void update(ParkingFloor floor, int availableSpots) {
        System.out.println("Floor " + floor.getFloorId() + ": " + availableSpots + " spots available");
    }
}

State Pattern

Problem it solves: An object has wildly different behavior depending on its current state. Without State pattern, you end up with massive if/else chains that are impossible to maintain.

💡
The "before/after" comparison

Without State pattern: one class with 5 states × 5 actions = 25 if/else branches, often with illegal state exceptions scattered everywhere. With State pattern: each state class handles exactly the actions valid in that state. Clean, extensible, easy to add new states.

JavaState Pattern — Vending Machine Skeleton
// State interface — every possible action in the machine
public interface VendingMachineState {
    void selectItem(String itemCode);
    void insertCoin(Coin coin);
    void dispenseItem();
    void cancelTransaction();
}

// Idle state — only selectItem is valid
public class IdleState implements VendingMachineState {
    private final VendingMachine machine;

    public void selectItem(String itemCode) {
        if (machine.getInventory().hasItem(itemCode)) {
            machine.setSelectedItem(itemCode);
            machine.setState(machine.getItemSelectedState()); // transition
        } else {
            System.out.println("Item not available");
        }
    }

    // All other actions are INVALID in IDLE state
    public void insertCoin(Coin coin)   { System.out.println("Select an item first"); }
    public void dispenseItem()            { System.out.println("Select an item first"); }
    public void cancelTransaction()       { System.out.println("Nothing to cancel"); }
}

Factory Method

Problem it solves: The caller needs an object but shouldn't be coupled to the exact class being instantiated. The factory decides what to create.

JavaFactory Method — Vehicle Creation
public class VehicleFactory {
    public static Vehicle create(VehicleType type, String licensePlate) {
        return switch (type) {
            case CAR      -> new Car(licensePlate);
            case BIKE     -> new Bike(licensePlate);
            case TRUCK    -> new Truck(licensePlate);
            case ELECTRIC -> new ElectricCar(licensePlate);
            default       -> throw new IllegalArgumentException("Unknown vehicle type");
        };
    }
}

Template Method

Problem it solves: Multiple classes follow the same overall algorithm structure, but certain steps vary. The parent class defines the skeleton; subclasses override specific steps.

JavaTemplate Method — Game Loop
public abstract class Game {
    // Template method — fixed algorithm skeleton
    public final void play() {
        initialize();       // step 1 — setup
        while (!isOver()) { // step 2 — loop until done
            takeTurn();
        }
        printWinner();       // step 3 — result
    }

    // Abstract steps — each game implements differently
    protected abstract void initialize();
    protected abstract boolean isOver();
    protected abstract void takeTurn();
    protected abstract void printWinner();
}

⚠️ Common LLD Mistakes to Avoid

Mistake 1 — Anemic Domain Model

Classes that are just bags of getters/setters with zero behavior. The logic ends up in one giant service class instead.

❌ Anemic (Wrong)

Java
// Logic leaks OUT of the entity
boolean available =
  parkingService.isSpotAvailable(spot);

// Service does everything
parkingService.assignVehicleToSpot(
  vehicle, spot);

✅ Rich Domain (Correct)

Java
// Logic lives IN the entity
boolean available =
  spot.isAvailable();

// Entity knows its own rules
spot.assignVehicle(vehicle);

Mistake 2 — Everything Public

Fields should be private. Expose behavior, not data. If you're making a field public for testing, that's a design smell.

🚫
public int availableSpots; — direct field access breaks encapsulation

Mistake 3 — God Class

ParkingLotManager should orchestrate, not implement everything. If it has 20+ methods, you have a God class.

🚫
One class doing payment, ticketing, floor management, AND spot assignment.

Mistake 4 — Strings Instead of Enums

Never use "HOURLY" when you mean FeeType.HOURLY. Strings are typo-prone and make switch statements fragile.

⚠️
Always define enums for any fixed set of values. The interviewer notices this.

Mistake 5 — Concrete Dependencies

Always depend on interfaces, not concrete classes.

🚫
private MySQLPaymentProcessor p; → should be private PaymentProcessor p;

🗣️ How to Talk While Coding

The Interviewer Cannot Read Your Mind

A senior engineer who codes in silence while making great decisions gets a lower score than an engineer who verbalizes average decisions well. Narrate your reasoning constantly.

Exact Phrases to Use

When making a class abstract:
"I'm making Vehicle abstract because Car, Bike, and Truck all share attributes like license plate and vehicle type, but they'll have different valid spot types. There's no such thing as a generic Vehicle — you always park a specific kind."
When applying Strategy pattern:
"I'll use Strategy pattern here for fee calculation — the algorithm varies by vehicle type AND might change in the future. Maybe we add monthly passes. Strategy lets me add MonthlyPassStrategy without touching any existing code."
When choosing interface over abstract class:
"I'm using an interface here rather than an abstract class because there's no shared implementation — just a contract. Multiple unrelated classes might implement this, and Java doesn't allow multiple inheritance of abstract classes."
When handling an edge case:
"Before I move on — let me think about what happens if the parking lot is full when a vehicle tries to enter. The EntranceGate.parkVehicle() method should check availability first and throw a checked exception or return an Optional. Let me handle that..."
When offering an extension point:
"Right now I've hardcoded 3 spot types. To make this extensible, I could make SpotType an interface so new types like EV charging spots can be added by a new class without changing existing code — that's Open/Closed Principle."

// Abstract Vehicle — no such thing as a "generic vehicle" that parks
public abstract class Vehicle {
    private final String      licensePlate;
    private final VehicleType type;

    protected Vehicle(String licensePlate, VehicleType type) {
        this.licensePlate = licensePlate;
        this.type          = type;
    }

    public String      getLicensePlate() { return licensePlate; }
    public VehicleType getType()         { return type; }

    // Each subclass declares which spot types it can use
    public abstract List<SpotType> getCompatibleSpotTypes();
}

public class Car extends Vehicle {
    public Car(String plate) { super(plate, VehicleType.CAR); }

    public List<SpotType> getCompatibleSpotTypes() {
        return List.of(SpotType.COMPACT, SpotType.REGULAR);
    }
}

public class Bike extends Vehicle {
    public Bike(String plate) { super(plate, VehicleType.BIKE); }

    public List<SpotType> getCompatibleSpotTypes() {
        return List.of(SpotType.COMPACT);
    }
}

public class Truck extends Vehicle {
    public Truck(String plate) { super(plate, VehicleType.TRUCK); }

    public List<SpotType> getCompatibleSpotTypes() {
        return List.of(SpotType.LARGE);
    }
}

// ParkingSpot — knows its own availability
public class ParkingSpot {
    private final String   spotId;
    private final SpotType spotType;
    private       Vehicle  parkedVehicle; // null when empty

    public ParkingSpot(String spotId, SpotType spotType) {
        this.spotId   = spotId;
        this.spotType = spotType;
    }

    // Rich domain model — spot knows its own rules
    public boolean isAvailable()  { return parkedVehicle == null; }
    public  SpotType getSpotType() { return spotType; }
    public  String   getSpotId()   { return spotId; }

    public void assignVehicle(Vehicle vehicle) {
        if (!isAvailable()) throw new IllegalStateException("Spot already occupied: " + spotId);
        this.parkedVehicle = vehicle;
    }

    public void removeVehicle() {
        if (isAvailable()) throw new IllegalStateException("Spot is already empty: " + spotId);
        this.parkedVehicle = null;
    }
}

// Ticket — issued on vehicle entry
public class Ticket {
    private final String        ticketId;
    private final Vehicle       vehicle;
    private final ParkingSpot   spot;
    private final LocalDateTime issuedTime;
    private       TicketStatus  status;

    public Ticket(Vehicle vehicle, ParkingSpot spot) {
        this.ticketId   = UUID.randomUUID().toString();
        this.vehicle    = vehicle;
        this.spot       = spot;
        this.issuedTime = LocalDateTime.now();
        this.status     = TicketStatus.ACTIVE;

    public String        getTicketId()   { return ticketId; }
    public Vehicle       getVehicle()    { return vehicle; }
    public ParkingSpot   getSpot()       { return spot; }
    public LocalDateTime getIssuedTime() { return issuedTime; }
    public TicketStatus  getStatus()     { return status; }
    public void           markPaid()     { this.status = TicketStatus.PAID; }
}

public class ParkingFloor implements ParkingObservable {

    private final List<ParkingSpot>     spots      = new ArrayList<>();
    private final List<ParkingObserver>  observers  = new ArrayList<>();

    public ParkingFloor(String floorId) { this.floorId = floorId; }

    public void addSpot(ParkingSpot spot)    { spots.add(spot); }
    public String getFloorId()               { return floorId; }
    public void addObserver(ParkingObserver obs) { observers.add(obs); }

    public void notifyObservers() {
        long avail = spots.stream().filter(ParkingSpot::isAvailable).count();
        observers.forEach(obs -> obs.update(floorId, avail));
    }

    // Find first available spot for a vehicle type
    public Optional<ParkingSpot> findAvailableSpot(Vehicle vehicle) {
        return spots.stream()
            .filter(ParkingSpot::isAvailable)
            .filter(s -> vehicle.getCompatibleSpotTypes().contains(s.getSpotType()))
            .findFirst();
    }
}

// STEP 5: MANAGER / ORCHESTRATOR (Singleton)
// ═══════════════════════════════════════════

public class ParkingLot {
    private static volatile ParkingLot     instance;
    private final String                   name;
    private final List<ParkingFloor>       floors    = new ArrayList<>();
    private final Map<String, Ticket>      activeTickets = new HashMap<>();

    private ParkingLot(String name) { this.name = name; }

    public static ParkingLot getInstance(String name) {
        if (instance == null) {
            synchronized (ParkingLot.class) {
                if (instance == null) instance = new ParkingLot(name);
            }
        }
        return instance;
    }

    public void addFloor(ParkingFloor floor) { floors.add(floor); }

    // Entry — find a spot and issue ticket
    public Ticket parkVehicle(Vehicle vehicle) {
        for (ParkingFloor floor : floors) {
            Optional<ParkingSpot> spot = floor.findAvailableSpot(vehicle);
            if (spot.isPresent()) {
                spot.get().assignVehicle(vehicle);           // occupy the spot
                Ticket ticket = new Ticket(vehicle, spot.get());
                activeTickets.put(ticket.getTicketId(), ticket); // track it
                floor.notifyObservers();                     // update display board
                return ticket;
            }
        }
        throw new RuntimeException("Parking lot is full for vehicle type: " + vehicle.getType());
    }

    // Exit — calculate fee, process payment, free spot
    public Payment unparkVehicle(String ticketId, FeeCalculationStrategy feeStrategy,
                                  PaymentMode paymentMode) {
        Ticket ticket = activeTickets.get(ticketId);
        if (ticket == null) throw new IllegalArgumentException("Invalid ticket: " + ticketId);
        if (ticket.getStatus() == TicketStatus.PAID) throw new IllegalStateException("Ticket already paid");

        double fee     = feeStrategy.calculateFee(ticket); // Strategy pattern
        Payment payment = new Payment(fee, paymentMode);

        ticket.getSpot().removeVehicle();  // free the spot
        ticket.markPaid();                 // mark ticket as paid
        activeTickets.remove(ticketId);   // remove from active

        // Notify display board — spot is now free
        floors.stream()
              .filter(f -> f.findAvailableSpot(ticket.getVehicle()).isPresent())
              .forEach(ParkingFloor::notifyObservers);

        return payment;
    }
}

public class Board {
    private final Cell[][] grid = new Cell[8][8];

    public Board() {
        for (int r = 0; r < 8; r++)
            for (int c = 0; c < 8; c++)
                grid[r][c] = new Cell(r, c);
        initializePieces();
    }

    public Cell getCell(int row, int col) {
        if (row < 0 || row > 7 || col < 0 || col > 7)
            throw new IllegalArgumentException("Out of board: " + row + "," + col);
        return grid[row][col];
    }

    public boolean isValid(int row, int col) {
        return row >= 0 && row < 8 && col >= 0 && col < 8;
    }

    private void initializePieces() {
        // White pieces on rows 0-1, Black on rows 6-7
        placePiecesForColor(PieceColor.WHITE, 0, 1);
        placePiecesForColor(PieceColor.BLACK, 7, 6);
    }

    private void placePiecesForColor(PieceColor color, int backRow, int pawnRow) {
        grid[backRow][0].setPiece(new Rook(color));
        grid[backRow][1].setPiece(new Knight(color));
        grid[backRow][2].setPiece(new Bishop(color));
        grid[backRow][3].setPiece(new Queen(color));
        grid[backRow][4].setPiece(new King(color));
        grid[backRow][5].setPiece(new Bishop(color));
        grid[backRow][6].setPiece(new Knight(color));
        grid[backRow][7].setPiece(new Rook(color));
        for (int c = 0; c < 8; c++) grid[pawnRow][c].setPiece(new Pawn(color));
    }

    public void movePiece(Cell src, Cell dst) {
        dst.setPiece(src.getPiece()); // move piece to destination
        src.setPiece(null);            // clear source cell

public abstract class Piece {
    protected final PieceColor color;
    protected boolean hasMoved = false; // important for pawn/king castling

    protected Piece(PieceColor color) { this.color = color; }

    public PieceColor getColor()   { return color; }
    public boolean    hasMoved()   { return hasMoved; }
    public void       setMoved()   { hasMoved = true; }

    // Each subclass MUST implement its own movement logic
    public abstract boolean canMove(Board board, Cell source, Cell destination);
    public abstract String  getSymbol();
}

// ═══════════════════ CONCRETE PIECES ═══════════════════

public class Rook extends Piece {
    public Rook(PieceColor color) { super(color); }

    public boolean canMove(Board board, Cell src, Cell dst) {
        // Rook moves in straight lines only (same row OR same column)
        if (src.getRow() != dst.getRow() && src.getCol() != dst.getCol()) return false;
        return isPathClear(board, src, dst);
    }

    // Helper: ensure no piece blocks the path
    private boolean isPathClear(Board board, Cell src, Cell dst) {
        int rowDir = Integer.signum(dst.getRow() - src.getRow());
        int colDir = Integer.signum(dst.getCol() - src.getCol());
        int r = src.getRow() + rowDir;
        int c = src.getCol() + colDir;
        while (r != dst.getRow() || c != dst.getCol()) {
            if (!board.getCell(r, c).isEmpty()) return false;
            r += rowDir; c += colDir;
        }
        // Destination must be empty or hold an enemy piece
        return dst.isEmpty() || dst.getPiece().getColor() != color;
    }

    public String getSymbol() { return color == PieceColor.WHITE ? "R" : "r"; }
}

public class Bishop extends Piece {
    public Bishop(PieceColor color) { super(color); }

    public boolean canMove(Board board, Cell src, Cell dst) {
        int rowDiff = Math.abs(dst.getRow() - src.getRow());
        int colDiff = Math.abs(dst.getCol() - src.getCol());
        if (rowDiff != colDiff) return false; // must be diagonal
        return isDiagonalPathClear(board, src, dst);
    }

    private boolean isDiagonalPathClear(Board board, Cell src, Cell dst) {
        int rowDir = Integer.signum(dst.getRow() - src.getRow());
        int colDir = Integer.signum(dst.getCol() - src.getCol());
        int r = src.getRow() + rowDir, c = src.getCol() + colDir;
        while (r != dst.getRow()) {
            if (!board.getCell(r, c).isEmpty()) return false;
            r += rowDir; c += colDir;
        }
        return dst.isEmpty() || dst.getPiece().getColor() != color;
    }

    public String getSymbol() { return color == PieceColor.WHITE ? "B" : "b"; }
}

public class Queen extends Piece {
    private final Rook   rook;
    private final Bishop bishop;

    public Queen(PieceColor color) {
        super(color);
        // Reuse Rook + Bishop logic — Queen = Rook + Bishop combined
        rook   = new Rook(color);
        bishop = new Bishop(color);
    }

    public boolean canMove(Board board, Cell src, Cell dst) {
        return rook.canMove(board, src, dst) || bishop.canMove(board, src, dst);
    }

    public String getSymbol() { return color == PieceColor.WHITE ? "Q" : "q"; }
}

public class Knight extends Piece {
    public Knight(PieceColor color) { super(color); }

    public boolean canMove(Board board, Cell src, Cell dst) {
        int rowDiff = Math.abs(dst.getRow() - src.getRow());
        int colDiff = Math.abs(dst.getCol() - src.getCol());
        // L-shape: (2+1) or (1+2) — Knight JUMPS, no path check needed
        if (!((rowDiff == 2 && colDiff == 1) || (rowDiff == 1 && colDiff == 2))) return false;
        return dst.isEmpty() || dst.getPiece().getColor() != color;
    }

    public String getSymbol() { return color == PieceColor.WHITE ? "N" : "n"; }
}

public class King extends Piece {
    public King(PieceColor color) { super(color); }

    public boolean canMove(Board board, Cell src, Cell dst) {
        int rowDiff = Math.abs(dst.getRow() - src.getRow());
        int colDiff = Math.abs(dst.getCol() - src.getCol());
        if (rowDiff > 1 || colDiff > 1) return false; // max 1 step
        return dst.isEmpty() || dst.getPiece().getColor() != color;
    }

    public String getSymbol() { return color == PieceColor.WHITE ? "K" : "k"; }
}

public class Pawn extends Piece {
    public Pawn(PieceColor color) { super(color); }

    public boolean canMove(Board board, Cell src, Cell dst) {
        int direction = (color == PieceColor.WHITE) ? 1 : -1; // white moves up, black moves down
        int rowDiff   = dst.getRow() - src.getRow();
        int colDiff   = Math.abs(dst.getCol() - src.getCol());

        // Forward move (same column, no capture)
        if (colDiff == 0 && dst.isEmpty()) {
            if (rowDiff == direction) return true; // single step forward
            if (rowDiff == 2 * direction && !hasMoved) { // 2 steps from start
                Cell middle = board.getCell(src.getRow() + direction, src.getCol());
                return middle.isEmpty();
            }
        }

        // Diagonal capture
        if (colDiff == 1 && rowDiff == direction && !dst.isEmpty()) {
            return dst.getPiece().getColor() != color; // must capture enemy
        }

        return false;
    }

    public String getSymbol() { return color == PieceColor.WHITE ? "P" : "p"; }
}

public class ChessGame {
    private final Board       board;
    private final Player[]    players;
    private       int          currentPlayerIndex = 0;
    private       GameStatus  status = GameStatus.ACTIVE;
    private final List<String> moveHistory = new ArrayList<>();

    public ChessGame(Player white, Player black) {
        this.board   = new Board();
        this.players = new Player[]{ white, black };
    }

    public boolean makeMove(int srcRow, int srcCol, int dstRow, int dstCol) {
        if (status != GameStatus.ACTIVE) throw new IllegalStateException("Game is over");

        Cell  src    = board.getCell(srcRow, srcCol);
        Cell  dst    = board.getCell(dstRow, dstCol);
        Piece piece  = src.getPiece();

        // Validate: piece exists and belongs to current player
        if (piece == null) { System.out.println("No piece at source"); return false; }
        if (piece.getColor() != players[currentPlayerIndex].getColor()) {
            System.out.println("Not your piece"); return false;
        }

        // Validate: piece's own movement rules
        if (!piece.canMove(board, src, dst)) {
            System.out.println("Invalid move for " + piece.getSymbol()); return false;
        }

        // Record and execute move
        String record = piece.getSymbol() + srcRow + srcCol + "->" + dstRow + dstCol;
        board.movePiece(src, dst);
        piece.setMoved();
        moveHistory.add(record);

        // Switch turns
        currentPlayerIndex = 1 - currentPlayerIndex;

        // Check game-over conditions (simplified)
        checkGameStatus();
        return true;
    }

    private void checkGameStatus() {
        // Simplified: check if opponent's King is captured
        PieceColor currentColor = players[currentPlayerIndex].getColor();
        boolean kingAlive = false;
        for (int r = 0; r < 8; r++) {
            for (int c = 0; c < 8; c++) {
                Piece p = board.getCell(r, c).getPiece();
                if (p instanceof King && p.getColor() == currentColor) kingAlive = true;
            }
        }
        if (!kingAlive) {
            status = (currentColor == PieceColor.WHITE) ? GameStatus.BLACK_WINS : GameStatus.WHITE_WINS;
            System.out.println("Game Over! " + status);
        }
    }
}

public class IdleState implements ElevatorState {
    public void addInternalRequest(Elevator elev, int floor) {
        elev.getRequests().add(floor);
        // Decide direction and start moving
        if (floor > elev.getCurrentFloor()) {
            elev.setState(elev.getMovingUpState());
        } else if (floor < elev.getCurrentFloor()) {
            elev.setState(elev.getMovingDownState());
        } else {
            elev.setState(elev.getDoorOpenState()); // already on requested floor
        }
    }
    public void openDoor(Elevator elev)  { elev.setState(elev.getDoorOpenState()); }
    public void closeDoor(Elevator elev) { System.out.println("Door already closed"); }
    public void moveUp(Elevator elev)    { System.out.println("No requests — staying idle"); }
    public void moveDown(Elevator elev)  { System.out.println("No requests — staying idle"); }
    public ElevatorStatus getStatus()   { return ElevatorStatus.IDLE; }
}

public class MovingUpState implements ElevatorState {
    public void addInternalRequest(Elevator elev, int floor) {
        if (floor > elev.getCurrentFloor()) {
            elev.getRequests().add(floor); // only add floors in current direction
        } else {
            elev.getPendingRequests().add(floor); // queue for after direction reversal
        }
    }
    public void moveUp(Elevator elev) {
        if (!elev.getRequests().isEmpty()) {
            int nextFloor = elev.getRequests().first(); // TreeSet — sorted ascending
            elev.setCurrentFloor(nextFloor);
            elev.getRequests().remove(nextFloor);
            System.out.println("Elevator arrived at floor: " + nextFloor);
            elev.setState(elev.getDoorOpenState());
        } else {
            elev.setState(elev.getIdleState()); // no more requests going up
        }
    }
    public void openDoor(Elevator elev)  { elev.setState(elev.getDoorOpenState()); }
    public void closeDoor(Elevator elev) { System.out.println("Door closed — moving"); }
    public void moveDown(Elevator elev)  { System.out.println("Can't move down while going up"); }
    public ElevatorStatus getStatus()   { return ElevatorStatus.MOVING_UP; }
}

    public TreeSet<Integer> getRequests()        { return requests; }
    public TreeSet<Integer> getPendingRequests()  { return pendingRequests; }
    public ElevatorState getIdleState()      { return idleState; }

public class ElevatorController {

    private final List<Elevator> elevators;

    private ElevatorController(int numElevators) {
        elevators = new ArrayList<>();
        for (int i = 1; i <= numElevators; i++) elevators.add(new Elevator(i));
    }

    public static ElevatorController getInstance(int n) {
        if (instance == null) synchronized (ElevatorController.class) {
            if (instance == null) instance = new ElevatorController(n);
        }
        return instance;
    }

    // Strategy: select nearest elevator that's idle or moving in same direction
    public void requestElevator(int floor, Direction direction) {
        Elevator best = elevators.stream()
            .min(Comparator.comparingInt(e ->
                Math.abs(e.getCurrentFloor() - floor)))
            .orElseThrow();

        System.out.println("Dispatching elevator " + best.getId() +
                           " to floor " + floor);
        best.addRequest(floor);
    }

    public void stepAll() {
        elevators.forEach(Elevator::step);
    }

        int    roll    = dice.roll();

        System.out.printf("%s rolled %d → position %d",
                          current.getName(), roll, newPos);

            // Must land EXACTLY on 100 — overshoot = stay put

                int jumpDest = cell.getJumpDestination();
                System.out.printf(" → %s! Move to %d%n",
                                  cell.getJumpType(), jumpDest);
                current.setPosition(jumpDest);
            } else {
                System.out.println();
            }

            if (current.getPosition() >= board.getSize()) {
                winner = current;
            }
        }

        currentIndex = (currentIndex + 1) % players.size();
    }

public class Catalog {

    private final Map<String, List<BookItem>> itemsByIsbn   = new HashMap<>();

        booksByIsbn.put(book.getIsbn(), book);
        itemsByIsbn.computeIfAbsent(book.getIsbn(), k -> new ArrayList<>()).add(item);
    }

    public List<BookItem> searchByIsbn(String isbn) {
        return itemsByIsbn.getOrDefault(isbn, Collections.emptyList());
    }

    public List<BookItem> searchByTitle(String title) {
        return itemsByIsbn.values().stream()
            .flatMap(Collection::stream)
            .filter(i -> i.getBook().getTitle().equalsIgnoreCase(title))
            .collect(Collectors.toList());
    }

    public List<BookItem> searchByAuthor(String author) {
        return itemsByIsbn.values().stream()
            .flatMap(Collection::stream)
            .filter(i -> i.getBook().getAuthor().equalsIgnoreCase(author))
            .collect(Collectors.toList());
    }

    public Optional<BookItem> findAvailableCopy(String isbn) {
        return searchByIsbn(isbn).stream().filter(BookItem::isAvailable).findFirst();

public class Library {
    private static volatile Library instance;
    private final Catalog          catalog;
    private final FineCalculator   fineCalculator;
    private final Map<String, Member> members = new HashMap<>();
    private final Map<String, Queue<Member>> reservationQueue = new HashMap<>();
    private static final int LOAN_DAYS = 14;

    private Library() {
        this.catalog        = new Catalog();
        this.fineCalculator = new PerDayFineCalculator();
    }

    public static Library getInstance() {
        if (instance == null) synchronized (Library.class) {
            if (instance == null) instance = new Library();
        }
        return instance;
    }

    public void addMember(Member m) { members.put(m.getMemberId(), m); }
    public void addBook(Book book, BookItem item) { catalog.addBook(book, item); }

    public BorrowRecord borrowBook(String memberId, String isbn) {
        Member   member = members.get(memberId);
        if (member == null)        throw new IllegalArgumentException("Member not found");
        if (!member.canBorrow())   throw new IllegalStateException("Member cannot borrow");

        BookItem item = catalog.findAvailableCopy(isbn)
            .orElseThrow(() -> new RuntimeException("No available copy for ISBN: " + isbn));

        item.checkout(LOAN_DAYS);
        member.borrowItem(item);

        BorrowRecord record = new BorrowRecord(member, item);
        System.out.printf("✅ %s borrowed '%s' — due %s%n",
                          member.getName(), item.getBook().getTitle(), item.getDueDate());
        return record;
    }

    public BorrowRecord returnBook(String memberId, BorrowRecord record) {
        Member   member = members.get(memberId);
        BookItem item   = record.getBookItem();
        LocalDate today  = LocalDate.now();

        double fine = fineCalculator.calculateFine(item.getDueDate(), today);
        record.setReturn(today, fine);

        item.returnItem();
        member.returnItem(item);

        if (fine > 0) {
            System.out.printf("⚠️ Late return. Fine: $%.2f%n", fine);
        }

        // Observer pattern: notify next member in reservation queue
        Queue<Member> queue = reservationQueue.get(item.getBook().getIsbn());
        if (queue != null && !queue.isEmpty()) {
            Member nextMember = queue.poll();
            System.out.println("📢 Notifying " + nextMember.getName() +
                               ": Your reserved book is now available!");
        }

        return record;
    }

    public void reserveBook(String memberId, String isbn) {
        Member member = members.get(memberId);
        reservationQueue.computeIfAbsent(isbn, k -> new LinkedList<>()).add(member);
        member.addReservation(isbn);
        System.out.println("📌 " + member.getName() + " reserved ISBN: " + isbn);
    }
}

public class IdleState implements VendingMachineState {
    public void selectItem(VendingMachine m, String code) {
        if (!m.getInventory().hasItem(code)) {
            System.out.println("❌ Item not available: " + code);
            return;
        }
        m.setSelectedItem(code);
        m.setState(m.getItemSelectedState());
        System.out.println("Selected: " + m.getInventory().getItem(code).getName()
                         + " ($" + m.getInventory().getItem(code).getPrice() + ")");
    }
    public void insertCoin(VendingMachine m, Coin c)   { System.out.println("Select an item first"); }
    public void dispenseItem(VendingMachine m)          { System.out.println("Select an item first"); }
    public void cancelTransaction(VendingMachine m)     { System.out.println("Nothing to cancel"); }
    public String getStateName()                        { return "IDLE"; }
}

public class ItemSelectedState implements VendingMachineState {
    public void selectItem(VendingMachine m, String c) { System.out.println("Item already selected"); }

    public void insertCoin(VendingMachine m, Coin coin) {
        m.addCoin(coin);
        m.setState(m.getPaymentPendingState());
        System.out.printf("Inserted %s. Total: %d cents%n", coin, m.getInsertedAmount());
    }

    public void dispenseItem(VendingMachine m)         { System.out.println("Insert coins first"); }

    public void cancelTransaction(VendingMachine m) {
        m.setSelectedItem(null);
        m.setState(m.getIdleState());
        System.out.println("Cancelled. No coins to refund.");
    }

    public String getStateName() { return "ITEM_SELECTED"; }
}

public class PaymentPendingState implements VendingMachineState {
    public void selectItem(VendingMachine m, String c) { System.out.println("Payment in progress"); }

    public void insertCoin(VendingMachine m, Coin coin) {
        m.addCoin(coin);
        System.out.printf("Inserted %s. Total: %d cents%n", coin, m.getInsertedAmount());
    }

    public void dispenseItem(VendingMachine m) {
        Item item  = m.getInventory().getItem(m.getSelectedItemCode());
        int  price = (int) (item.getPrice() * 100); // convert to cents

        if (m.getInsertedAmount() < price) {
            System.out.printf("Insufficient funds. Need %d more cents.%n",
                             price - m.getInsertedAmount());
            return;
        }

        // Enough money — transition to dispensing
        m.setState(m.getDispensingState());
        m.getState().dispenseItem(m); // immediately trigger dispense in new state
    }

    public void cancelTransaction(VendingMachine m) {
        int refund = m.getInsertedAmount();
        m.resetTransaction();
        m.setState(m.getIdleState());
        System.out.println("Cancelled. Refunding " + refund + " cents.");
    }

    public String getStateName() { return "PAYMENT_PENDING"; }
}

public class DispensingState implements VendingMachineState {
    public void dispenseItem(VendingMachine m) {
        Item item  = m.getInventory().getItem(m.getSelectedItemCode());
        int  price = (int) (item.getPrice() * 100);
        int  change = m.getInsertedAmount() - price;

        item.dispense();
        System.out.println("🎉 Dispensing: " + item.getName());
        if (change > 0) System.out.println("💰 Change returned: " + change + " cents");

        m.resetTransaction();
        m.setState(m.getIdleState());
    }

    public void selectItem(VendingMachine m, String c) { System.out.println("Machine busy"); }
    public void insertCoin(VendingMachine m, Coin c)   { System.out.println("Machine busy"); }
    public void cancelTransaction(VendingMachine m)     { System.out.println("Cannot cancel — dispensing"); }
    public String getStateName()                        { return "DISPENSING"; }
}

public class VendingMachine {

    private final VendingMachineState paymentPendingState  = new PaymentPendingState();
    private final VendingMachineState dispensingState      = new DispensingState();

    public void selectItem(String code)  { state.selectItem(this, code); }
    public void insertCoin(Coin coin)    { state.insertCoin(this, coin); }

⏱️ The 45-Minute LLD Interview Timeline

Minutes 0–5: Requirements Gathering

Ask your 3–4 clarifying questions. Write down use cases as a numbered list where the interviewer can see. Confirm scope boundaries — explicitly say what's OUT of scope.

🎯
What interviewers watch:

Do you ask the RIGHT questions, or generic ones? "Should I handle concurrent access?" → weak. "Should a vehicle be able to reserve a spot in advance, or only on-arrival?" → strong.

Minutes 5–12: Design Discussion (No Code)

Name your entities and explain why they exist. Define relationships aloud. Identify 1–2 design patterns that apply and justify them. Sketch the class diagram.

⚠️
Most common failure point

Jumping to code before minute 12. This signals to the interviewer that you can't design — you can only implement. The design discussion IS the interview.

Minutes 12–35: Code

Follow the coding order: enums → models → interfaces → implementations → manager. Narrate every significant decision as you write it.

Coding Order Checklist

  1. ✅ Enums (VehicleType, SpotType, etc.)
  2. ✅ Core model classes
  3. ✅ Interfaces and abstract classes
  4. ✅ Concrete implementations
  5. ✅ Manager/Singleton/Orchestrator
  6. ✅ Demo main method
Minutes 35–42: Edge Cases + Walk Through

Run through one complete use case step-by-step in your code. Name 3–4 edge cases. Offer one extension scenario to show forward-thinking.

Edge Cases to Mention

  • System full / empty scenarios
  • Same operation submitted twice
  • Type mismatch (wrong vehicle for spot)
  • Concurrent access (thread safety)

Extension Scenarios

  • New payment method added
  • New vehicle/item type added
  • Analytics/reporting requirement
  • Multi-location / multi-building
Minutes 42–45: Questions for Interviewer
  • "What would you extend in this design first?"
  • "Is there a pattern you'd have applied differently?"
  • "What scale does this typically run at in your system?"
💡
Why this matters

Asking good questions at the end signals intellectual curiosity and that you're thinking beyond the immediate task — a strong senior engineer signal.

🗣️ Phrases That Signal Senior Thinking

These phrases do double duty

They demonstrate design knowledge AND they give you time to think. Practice saying them until they're reflexive.

When starting every problem:
"Before I jump into code, let me clarify a few requirements to make sure I'm designing the right thing..."
When identifying an abstraction:
"I see Vehicle is a natural abstraction here — there's no such thing as a generic vehicle that parks. You always park a Car or a Bike, each with different rules."
When applying Strategy pattern:
"I'm applying Strategy for fee calculation because the algorithm varies by vehicle type AND this is likely to change — maybe we add monthly passes next quarter. Strategy lets me extend without touching existing code — that's Open/Closed Principle."
When choosing Composition over Inheritance:
"I could inherit here but I'll use composition instead — it's more flexible and avoids the tight coupling that deep inheritance hierarchies create."
When justifying Singleton:
"I'll use Singleton for ParkingLotManager because having two managers would mean split inventories — that would be a bug, not a feature."
When handling an edge case mid-code:
"Let me pause — what happens if the lot is full when a vehicle tries to enter? I should handle that here rather than letting it throw a NullPointerException silently..."
When offering an extension point:
"This could be made more extensible by extracting this fee algorithm as a strategy — then adding a weekly pass later is just a new class, no changes to existing code."

CCP — Common Closure Principle Classes that change together belong together. This is SRP at the package level. A feature change should touch one package, not ten
CRP — Common Reuse Principle Don't force users of a package to depend on things they don't use. Interface Segregation Principle at the package level
ADP — Acyclic Dependencies Principle No cycles in the package dependency graph. Package A depends on B depends on A → can't change either independently
SDP — Stable Dependencies Principle Depend in the direction of stability. Depend on things that don't change. Volatile modules shouldn't be depended on by stable ones
SAP — Stable Abstractions Principle The more stable a component, the more abstract it should be. Your core domain model (most stable) should be all interfaces/abstractions

Cohesion vs Coupling — The Fundamental Tension

Every architectural decision is ultimately about trading off cohesion and coupling. Architects maximize cohesion within components and minimize coupling between them.

High Cohesion ✅
  • Classes/methods in a module do one related thing
  • Changes are localized — edit one place
  • Easy to understand, test, and reuse
  • Example: OrderRepository only does order persistence
Low Coupling ✅
  • Modules know as little about each other as possible
  • Changes in one module don't ripple to others
  • Communicate through stable interfaces/events
  • Example: Payment service emits event; Email service subscribes
💡
Architect's heuristic: If a change requires you to touch more than 2–3 files in more than 1 layer, your coupling is too tight. Refactor toward events, interfaces, or anti-corruption layers.

⚡ Concurrency Design Patterns

🎯
Why architects care: Concurrency bugs are the hardest bugs in production. Architects choose patterns that make concurrent code correct by construction, not by luck.

Thread-Safe Singleton — Double-Checked Locking

JavaThread-Safe Singleton — 3 Correct Ways
// ── Way 1: Enum (Bill Pugh — recommended for most cases) ────
public enum DatabasePool {
    INSTANCE;
    public Connection getConnection() { ... }
}
// JVM guarantees enum instances are created once; free serialization safety

// ── Way 2: Holder Pattern (lazy, no synchronization overhead) ─
public class ConfigManager {
    private ConfigManager() {}
    private static class Holder {
        static final ConfigManager INSTANCE = new ConfigManager();
    }
    public static ConfigManager getInstance() { return Holder.INSTANCE; }
}
// Class loading is thread-safe; Holder only loaded on first getInstance() call

// ── Way 3: Double-Checked Locking (when you can't use above) ─
public class CacheManager {
    private static volatile CacheManager instance; // volatile is REQUIRED

    public static CacheManager getInstance() {
        if (instance == null) {                        // first check (no lock)
            synchronized (CacheManager.class) {
                if (instance == null) {               // second check (with lock)
                    instance = new CacheManager();
                }
            }
        }
        return instance;
    }
}
// volatile prevents CPU instruction reordering during object construction

Producer-Consumer with BlockingQueue

JavaThread-Safe Task Queue
public class TaskProcessor {
    private final BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100);
    private final ExecutorService consumers = Executors.newFixedThreadPool(4);

    public void start() {
        for (int i = 0; i < 4; i++) {
            consumers.submit(() -> {
                while (!Thread.currentThread().isInterrupted()) {
                    try {
                        Task task = queue.take();  // blocks until task available
                        task.execute();
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                    }
                }
            });
        }
    }

    public void submit(Task task) throws InterruptedException {
        queue.put(task);  // blocks if queue full (backpressure!)
    }
}
// BlockingQueue.put() = natural backpressure; producer slows down when consumer is overwhelmed

Read-Write Lock — Optimizing Read-Heavy Data

JavaReentrantReadWriteLock Pattern
public class ConfigCache {
    private final Map<String, String> cache = new HashMap<>();
    private final ReadWriteLock lock = new ReentrantReadWriteLock();

    public String get(String key) {
        lock.readLock().lock();           // multiple threads can read concurrently
        try { return cache.get(key); }
        finally { lock.readLock().unlock(); }
    }

    public void refresh(Map<String, String> newConfig) {
        lock.writeLock().lock();          // exclusive write — all readers blocked
        try {
            cache.clear();
            cache.putAll(newConfig);
        } finally { lock.writeLock().unlock(); }
    }
}
// 1000 reads/sec, 1 write/hour → ReadWriteLock gives near-zero read contention

Immutability — The Ultimate Concurrency Pattern

JavaImmutable Value Object (Java Record)
// Immutable objects require ZERO synchronization — share freely across threads
public record Money(BigDecimal amount, Currency currency) {

    public Money {  // compact constructor for validation
        Objects.requireNonNull(amount, "amount required");
        Objects.requireNonNull(currency, "currency required");
        if (amount.scale() > 2) throw new IllegalArgumentException("Max 2 decimal places");
    }

    public Money add(Money other) {
        if (!currency.equals(other.currency)) throw new CurrencyMismatchException();
        return new Money(amount.add(other.amount), currency);  // returns new — never mutates
    }

    public Money multiply(int factor) {
        return new Money(amount.multiply(BigDecimal.valueOf(factor)), currency);
    }
}
// equals/hashCode/toString/accessors all generated by record — zero boilerplate

🚀 Performance Design Patterns

Object Pool Pattern

Reuse expensive-to-create objects (DB connections, thread pools, HTTP clients) instead of creating/destroying them on each request.

JavaGeneric Object Pool
public class ObjectPool<T> {
    private final BlockingQueue<T> pool;
    private final Supplier<T>        factory;
    private final Consumer<T>        reset;  // cleanup before reuse

    public ObjectPool(int size, Supplier<T> factory, Consumer<T> reset) {
        this.factory = factory;
        this.reset   = reset;
        pool = new ArrayBlockingQueue<>(size);
        IntStream.range(0, size).forEach(i -> pool.offer(factory.get()));
    }

    public T borrow() throws InterruptedException {
        return pool.poll(5, TimeUnit.SECONDS);  // wait up to 5s
    }

    public void returnToPool(T obj) {
        reset.accept(obj);   // clear state before reuse
        pool.offer(obj);
    }
}
// Usage: HikariCP, Apache Commons Pool, Netty's ByteBuf pool

Cache-Aside (Lazy Loading) Pattern

JavaCache-Aside with Spring + Redis
@Service
public class ProductService {
    private final ProductRepository  db;
    private final RedisTemplate<String, Product> redis;
    private static final Duration TTL = Duration.ofMinutes(15);

    public Product findById(UUID id) {
        String key = "product:" + id;

        // 1. Check cache
        Product cached = redis.opsForValue().get(key);
        if (cached != null) return cached;

        // 2. Cache miss — load from DB
        Product product = db.findById(id).orElseThrow();

        // 3. Populate cache
        redis.opsForValue().set(key, product, TTL);
        return product;
    }

    public void update(Product product) {
        db.save(product);
        redis.delete("product:" + product.getId());  // invalidate on write
    }
}
// vs @Cacheable annotation — manual gives control over TTL and invalidation strategy

Problem 7 — Rate Limiter (Token Bucket Algorithm)

💡
Used in: API gateways, DoS protection, fair use enforcement. Patterns: Strategy (swappable algorithms), Singleton (per-user state), Thread-safe state management.
JavaToken Bucket Rate Limiter
public class TokenBucketRateLimiter {
    private final int      capacity;       // max tokens (burst size)
    private final double   refillRate;     // tokens per second
    private double         tokens;
    private long           lastRefillTime;

    public TokenBucketRateLimiter(int capacity, double refillRate) {
        this.capacity       = capacity;
        this.refillRate     = refillRate;
        this.tokens         = capacity;
        this.lastRefillTime = System.nanoTime();
    }

    public synchronized boolean tryAcquire() {
        refill();
        if (tokens >= 1.0) {
            tokens--;
            return true;     // request allowed
        }
        return false;         // rate limit exceeded
    }

    private void refill() {
        long now     = System.nanoTime();
        double elapsed = (now - lastRefillTime) / 1_000_000_000.0;
        tokens         = Math.min(capacity, tokens + elapsed * refillRate);
        lastRefillTime = now;
    }
}

// ── Multi-user Rate Limiter ──────────────────────────────────────
@Component
public class RateLimiterService {
    // ConcurrentHashMap.computeIfAbsent = thread-safe per-user bucket creation
    private final ConcurrentHashMap<String, TokenBucketRateLimiter> limiters
        = new ConcurrentHashMap<>();

    public boolean isAllowed(String userId) {
        TokenBucketRateLimiter limiter = limiters.computeIfAbsent(
            userId,
            k -> new TokenBucketRateLimiter(100, 10.0) // 100 burst, 10/sec refill
        );
        return limiter.tryAcquire();
    }
}

// Spring filter usage:
if (!rateLimiter.isAllowed(userId)) {
    response.setStatus(429); // 429 Too Many Requests
    return;
}

Problem 8 — URL Shortener (TinyURL)

💡
Patterns: Strategy (encoding algorithm), Singleton (counter), Factory (URL validator). Key decisions: Base62 encoding, collision handling, redirect caching.
JavaURL Shortener — Core Design
public class UrlShortenerService {
    private static final String BASE62 = "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789";
    private final AtomicLong        counter = new AtomicLong(10_000_000L); // start high for 6-char URLs
    private final Map<String, String>  shortToLong = new ConcurrentHashMap<>();
    private final Map<String, String>  longToShort = new ConcurrentHashMap<>();

    public String shorten(String longUrl) {
        // Idempotent: same long URL always returns same short code
        return longToShort.computeIfAbsent(longUrl, url -> {
            String code = toBase62(counter.getAndIncrement());
            shortToLong.put(code, url);
            return code;
        });
    }

    public String resolve(String code) {
        String url = shortToLong.get(code);
        if (url == null) throw new NotFoundException("Short code not found: " + code);
        return url;
    }

    private String toBase62(long num) {
        StringBuilder sb = new StringBuilder();
        while (num > 0) {
            sb.append(BASE62.charAt((int)(num % 62)));
            num /= 62;
        }
        return sb.reverse().toString();
    }
}

// For production: replace in-memory maps with Redis (shortToLong TTL = expiry)
// counter = Redis INCR (atomic, distributed)
// Redirect: HTTP 301 (permanent, browser caches) vs 302 (temporary, analytics captured)
