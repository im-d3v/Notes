Union Find (Disjoint Set Union)

🧠  Mental Model: Imagine a social network. Each person starts in their own group. When two people connect, their groups merge. Ask "are A and B in the same group?" in near-constant time — even after millions of merges.
Recognition Signals
"connected components" "same group/cluster" "union of sets" "cycle in undirected graph" "number of islands" "friend circles"

Key tell: you're repeatedly asked "do X and Y belong together?" or "how many separate groups exist?"

Complexity
Find ≈ O(1) amortized
Union ≈ O(1) amortized
Space O(N)
When NOT to Use

Directed graphs — use Topological Sort or DFS instead. DSU only works on undirected connections.

Java Union Find — Path Compression + Union by Rank 
class UnionFind {
    private int[] parent, rank;

    UnionFind(int n) {
        parent = new int[n];
        rank   = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i; // each node is its own root
    }

    // Path compression — flatten tree on every find
    public int find(int x) {
        if (parent[x] != x)
            parent[x] = find(parent[x]); // recursively compress
        return parent[x];
    }

    // Union by rank — always attach smaller tree under larger
    public boolean union(int x, int y) {
        int px = find(x), py = find(y);
        if (px == py) return false; // already connected — CYCLE detected
        if (rank[px] < rank[py]) { int tmp = px; px = py; py = tmp; }
        parent[py] = px;
        if (rank[px] == rank[py]) rank[px]++;
        return true;
    }

    public boolean connected(int x, int y) {
        return find(x) == find(y);
    }
}

// Usage pattern — number of components
int components = n;
for (int[] edge : edges) {
    if (uf.union(edge[0], edge[1])) components--; // successful merge
}
⚡ Common Trap

Using DSU on a directed graph. Union Find only models undirected edges. If A→B exists, you cannot blindly call union(A,B) — that would mean B also connects back to A, which may be false. Use BFS/DFS or Kahn's algorithm for directed graphs.

Topological Sort (Kahn's Algorithm)

🧠  Mental Model: Before you can eat a meal, you need ingredients. Before ingredients, you need a recipe. Topological sort finds the legal ordering of tasks with dependencies. If any cycle exists (A depends on B, B depends on A) — ordering is impossible.
Recognition Signals
"course prerequisites" "task scheduling" "dependency ordering" "directed acyclic graph" "build order" "alien dictionary"
Cycle Detection Rule
🚨
If processed nodes < total nodes → cycle exists.

Some nodes never reach in-degree 0, so they're never added to queue.

Java Kahn's Algorithm (BFS-based Topological Sort) 
public int[] topoSort(int numNodes, int[][] prerequisites) {
    List<List<Integer>> adj    = new ArrayList<>();
    int[] inDegree = new int[numNodes];

    for (int i = 0; i < numNodes; i++) adj.add(new ArrayList<>());
    for (int[] pre : prerequisites) {
        adj.get(pre[1]).add(pre[0]);  // pre[1] → pre[0]
        inDegree[pre[0]]++;
    }

    Queue<Integer> queue = new LinkedList<>();
    for (int i = 0; i < numNodes; i++)
        if (inDegree[i] == 0) queue.offer(i); // seed: all "free" nodes

    int[] order   = new int[numNodes];
    int  processed = 0;

    while (!queue.isEmpty()) {
        int node = queue.poll();
        order[processed++] = node;
        for (int next : adj.get(node)) {
            if (--inDegree[next] == 0) queue.offer(next); // dependency resolved
        }
    }

    return processed == numNodes ? order : new int[0]; // [] = cycle detected
}
⚡ Common Trap

Getting edge direction backwards. prerequisites[i] = [a, b] means "to take course a, you must first take b" → edge is b → a, so in-degree of a increases. Drawing a quick diagram before coding saves 10 minutes of debugging.

  • 🔴LC #269 — Alien Dictionary— build graph from char ordering, then topo sort

Trie (Prefix Tree)

🧠  Mental Model: A dictionary organised by first letter, then second letter, then third… Like a filing cabinet where the folder at each level corresponds to one character. All words sharing a prefix share the same root path.
Recognition Signals
"starts with prefix" "autocomplete" "word search in board" "longest common prefix" "dictionary / spell check"
Trie vs HashMap
NeedUse
Exact matchHashMap
Prefix matchTrie
All words with prefixTrie
Visual — Words: "cat", "cap", "car"
(root) └── c └── a ├── t ← isEnd=true ("cat") ├── p ← isEnd=true ("cap") └── r ← isEnd=true ("car")

startsWith("ca") → walk c→a, return true. All 3 words share this path.

Java Trie — Insert + Search + StartsWith 
class Trie {
    private TrieNode root = new TrieNode();

    static class TrieNode {
        TrieNode[] children = new TrieNode[26]; // lowercase a-z
        boolean isEnd = false;
    }

    public void insert(String word) {
        TrieNode cur = root;
        for (char c : word.toCharArray()) {
            int idx = c - 'a';
            if (cur.children[idx] == null)
                cur.children[idx] = new TrieNode();
            cur = cur.children[idx];
        }
        cur.isEnd = true;
    }

    public boolean search(String word) {
        TrieNode node = walk(word);
        return node != null && node.isEnd; // must reach end AND isEnd
    }

    public boolean startsWith(String prefix) {
        return walk(prefix) != null; // just needs to reach last char
    }

    private TrieNode walk(String s) {
        TrieNode cur = root;
        for (char c : s.toCharArray()) {
            int idx = c - 'a';
            if (cur.children[idx] == null) return null;
            cur = cur.children[idx];
        }
        return cur;
    }
}
⚡ Common Trap

Confusing search() and startsWith(). search("cat") requires reaching node 't' AND isEnd == true. startsWith("ca") only needs to reach node 'a'. Forgetting the isEnd check causes "cap" to incorrectly match a search for "ca".

Two Heaps — Median Maintenance

🧠  Mental Model: Split a sorted list at the middle. Left half → max-heap (quickly get the largest of the lower half). Right half → min-heap (quickly get the smallest of the upper half). The median lives at the boundary between the two heaps.
Recognition Signals
"median of data stream" "continuously inserted numbers" "sliding window median" "find median at each step"
💡
Key invariant: |maxHeap.size - minHeap.size| ≤ 1

Java Two Heaps — MedianFinder 
class MedianFinder {
    // maxHeap → stores lower half  (top = largest of lower)
    // minHeap → stores upper half  (top = smallest of upper)
    PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();

    public void addNum(int num) {
        maxHeap.offer(num);                        // 1. always add to maxHeap first
        minHeap.offer(maxHeap.poll());            // 2. balance: push max of lower → upper
        if (minHeap.size() > maxHeap.size())     // 3. keep maxHeap >= minHeap in size
            maxHeap.offer(minHeap.poll());
    }

    public double findMedian() {
        if (maxHeap.size() > minHeap.size())
            return maxHeap.peek();                // odd count: median is top of maxHeap
        return (maxHeap.peek() + minHeap.peek()) / 2.0; // even: average of two tops
    }
}
⚡ Common Trap

Forgetting to use 2.0 (not 2) for the division. Integer division truncates: (5+8)/2 = 6, not 6.5. Always cast or use 2.0 when the return type is double.

K-Way Merge

🧠  Mental Model: You have K sorted queues. You always want the globally smallest next element. A min-heap acts as a "smart router" — it always points you to which queue holds the next smallest value. Pop one, push its successor.
Recognition Signals
"merge K sorted lists" "Kth smallest across arrays" "sorted matrix Kth element" "multiple sorted streams"
Complexity
Time O(N log K)
Space O(K)

K = number of lists. N = total elements. The heap never holds more than K elements.

Java K-Way Merge — Merge K Sorted Lists 
public ListNode mergeKLists(ListNode[] lists) {
    // Heap stores [value, listIndex, node] — ordered by value
    PriorityQueue<int[]> heap = new PriorityQueue<>((a, b) -> a[0] - b[0]);

    // Seed heap with head of each list
    for (int i = 0; i < lists.length; i++)
        if (lists[i] != null)
            heap.offer(new int[]{ lists[i].val, i, 0 });

    ListNode dummy = new ListNode(0), cur = dummy;

    while (!heap.isEmpty()) {
        int[] top = heap.poll();              // globally smallest node
        ListNode node = getNode(lists, top);  // retrieve actual node
        cur.next = node;
        cur = cur.next;
        if (node.next != null)             // push next from same list
            heap.offer(new int[]{ node.next.val, top[1], top[2] + 1 });
    }
    return dummy.next;
}
⚡ Common Trap

Seeding the heap without checking if a list is null or empty. If lists[i] == null, calling lists[i].val throws NPE. Always guard with a null check when initializing the heap.

Modified Binary Search

🧠  Mental Model: Classic binary search requires a perfectly sorted array. Modified binary search works even when the array is rotated or partially sorted — because at least one half is always fully sorted. Use the sorted half to decide which side to eliminate.
Recognition Signals
"rotated sorted array" "find peak element" "search in matrix" "minimum in rotated array" O(log N) required
Decision Rule — Which Half is Sorted?
🔑
if arr[mid] >= arr[left] → left half is sorted
else → right half is sorted
Then check if target falls inside the sorted half.
Java Binary Search in Rotated Sorted Array — LC #33 
public int search(int[] nums, int target) {
    int left = 0, right = nums.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2; // avoid overflow

        if (nums[mid] == target) return mid;

        // Left half [left..mid] is fully sorted
        if (nums[left] <= nums[mid]) {
            if (nums[left] <= target && target < nums[mid])
                right = mid - 1;   // target in left half
            else
                left  = mid + 1;   // target in right half
        } else { // Right half [mid..right] is fully sorted
            if (nums[mid] < target && target <= nums[right])
                left  = mid + 1;   // target in right half
            else
                right = mid - 1;   // target in left half
        }
    }
    return -1;
}
⚡ Common Trap

Using (left + right) / 2 for mid. With large indices this causes integer overflow. Always use left + (right - left) / 2. It's a one-character difference but a critical bug in production with arrays over ~2 billion elements.

  • 🟡LC #240 — Search a 2D Matrix II— start top-right: go left if too big, down if too small

Cyclic Sort

🧠  Mental Model: If numbers are in range [1..N], each number has a correct home index: number k belongs at index k-1. Walk the array — whenever a number is not at its home, swap it there. After one pass, every misplaced number reveals itself.
Recognition Signals
"numbers in range [1..n]" "find missing number" "find duplicate" "all numbers from 1 to n" "in-place O(1) space"
Interactive — Sort [3,1,5,4,2]

Java Cyclic Sort + Find Missing Number 
// Step 1: Place each number at its correct index
public void cyclicSort(int[] nums) {
    int i = 0;
    while (i < nums.length) {
        int correct = nums[i] - 1; // where nums[i] should live
        if (nums[i] != nums[correct]) {
            // Swap nums[i] to its correct position
            int tmp = nums[correct];
            nums[correct] = nums[i];
            nums[i] = tmp;
        } else {
            i++; // already at correct index (or duplicate) — move on
        }
    }
}

// Step 2: Find the first index where nums[i] != i+1
public int findMissing(int[] nums) {
    cyclicSort(nums);
    for (int i = 0; i < nums.length; i++)
        if (nums[i] != i + 1) return i + 1;
    return nums.length + 1; // missing number is n+1
}
⚡ Common Trap

Using nums[i] != i+1 as the swap condition causes an infinite loop when duplicates exist. If both positions hold the same value, you'll keep swapping forever. Always guard: if (nums[i] != nums[correct]) — compare by value at destination, not index.

Java XOR Templates — Single Number & Two Non-Repeating 
// ── Template 1: Single number in array where all others appear twice
public int singleNumber(int[] nums) {
    int result = 0;
    for (int n : nums) result ^= n; // pairs cancel, single survives
    return result;
}

// ── Template 2: Two unique numbers (LC #260)
public int[] singleNumberIII(int[] nums) {
    int xor = 0;
    for (int n : nums) xor ^= n; // xor = a ^ b

    // Find rightmost bit where a and b differ
    int diffBit = xor & (-xor);

    int a = 0, b = 0;
    for (int n : nums) {
        if ((n & diffBit) != 0) a ^= n; // group with diffBit set
        else                      b ^= n; // group without diffBit
    }
    return new int[]{ a, b };
}

// ── Template 3: Appears three times, find the one that appears once (LC #137)
public int singleNumberII(int[] nums) {
    int ones = 0, twos = 0;
    for (int n : nums) {
        ones = (ones ^ n) & ~twos;
        twos = (twos ^ n) & ~ones;
    }
    return ones; // bit counter trick — 3-state machine per bit
}
⚡ Common Trap

Trying to use basic XOR (Template 1) for LC #137 where elements appear 3 times. a^a^a = a, not 0 — triples don't cancel! You need the bit counter technique (Template 3) or count bits modulo 3 per position.

⚡ One-Line Cheat Sheet

13
Union Find

union() returns false? → cycle found. Count successful merges = n - components.

14
Topo Sort

Seed queue with in-degree 0. If processed < n → cycle exists → impossible ordering.

15
Trie

search() needs isEnd=true at last node. startsWith() just needs the path to exist.

16
Two Heaps

Always add to maxHeap first, then rebalance. maxHeap.size() ≥ minHeap.size() always.

17
K-Way Merge

Seed heap with all list heads. Pop min, push its .next. Heap size stays ≤ K throughout.

18
Modified BS

One half is always sorted. Check if target is in the sorted half. Always use left + (right-left)/2.

19
Cyclic Sort

num k → index k-1. Swap if misplaced. Guard nums[i] != nums[correct] to avoid infinite loop on duplicates.

20
Bitwise XOR

a^a=0, a^0=a. XOR all → pairs vanish. Use ones+twos trick when elements appear 3× not 2×.
